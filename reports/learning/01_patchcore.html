PatchCore — Towards Total Recall in Industrial Anomaly Detection

Roth et al., CVPR 2022 arXiv:2106.08265 본 연구 베이스라인 1순위

1. 왜 PatchCore가 필요했는가

산업 현장의 이상 탐지에는 두 가지 근본적 모순이 있습니다.

모순설명
① 정상 데이터는 많고, 이상 데이터는 거의 없다지도학습 불가 → unsupervised / one-class 패러다임
② "이상"의 종류를 사전 정의할 수 없다모델이 "정상이 아닌 모든 것"을 검출해야 함

PatchCore 이전의 PaDiM(2020)은 ImageNet pretrained 모델의 patch-level feature를 다변량 정규분포로 모델링했습니다. 하지만:

  • 분포 가정이 강함 — 실제 정상 분포가 정규분포가 아닐 수 있음
  • 메모리 비용이 폭증 — 위치마다 공분산 행렬 저장
PatchCore의 아이디어 분포 가정을 버리고 "정상 feature들의 큰 모음"을 그냥 들고 다니자. 그리고 테스트 시점에는 "이 patch가 정상 모음에서 얼마나 멀리 떨어져 있나?"만 측정.

2. 직관 — Memory Bank Metaphor

[학습]                              [추론]

  정상 이미지 100~1000장              테스트 이미지 1장
  ↓ ImageNet pretrained CNN          ↓ 같은 CNN
  ↓ 중간 layer feature 추출           ↓ 같은 feature
  ↓ patch 단위로 펼치기               ↓ patch 단위

  [📚 Memory Bank]                   각 patch에 대해:
  "정상 patch feature 모음"      ←→  Memory Bank에서 가장 가까운 patch
  (수십만~수백만 vector)             와의 거리 = anomaly score
핵심 비유 도서관에 "정상 책"이 가득 차 있는데, 새 책을 들고 와서 "이 책과 가장 비슷한 책이 도서관에 있는가?"를 묻는 것. 비슷한 책이 없으면 → 이상.

3. 알고리즘 (3단계)

  1. Feature 추출 (Pretrained Encoder)

    PatchCore는 ImageNet pretrained WideResNet-50을 그대로 씁니다 (학습 없음).

    • 입력 이미지 224×224
    • WideResNet의 layer 2 + layer 3 출력을 가져옴
      • layer 2 출력: shape (512, 28, 28) — 더 세밀
      • layer 3 출력: shape (1024, 14, 14) — 더 추상
    • layer 3을 upsample하여 28×28로 맞추고 채널 방향 concat → (1536, 28, 28)
    • spatial하게 펼치면 28×28 = 784개의 patch feature, 각각 1536-dim

    왜 layer 2+3? 너무 얕은 layer는 텍스처만 보고, 너무 깊은 layer는 의미만 봐서 structural anomaly를 놓침. 중간이 sweet spot.

  2. Coreset Subsampling — 메모리 압축

    정상 이미지 200장이면 memory bank에는 200 × 784 = 156,800개의 vector. 너무 많음. PatchCore는 greedy coreset selection으로 1%(~1,568개)만 유지:

    1. Memory bank M = ∅, 후보 풀 P = 전체 patch feature
2. 첫 vector를 P에서 무작위로 뽑아 M에 추가
3. 반복 (1%가 될 때까지):
     - P에서 "M의 모든 vector와 가장 멀리 떨어진" vector 선택
     - 그 vector를 M에 추가

    직관: "메모리 안에 다양성을 최대화하도록 뽑는다." 비슷한 vector는 1개만 남기고, 가능한 한 공간을 골고루 cover.

  3. 추론 — Nearest Neighbor 거리

    테스트 이미지가 들어오면:

    • 동일 feature 추출 → 784개 patch feature
    • 각 patch에 대해 memory bank에서 k-NN 거리 계산 (보통 k=1)

    Patch 단위 score: $s(x_{\text{patch}}) = \|f(x_{\text{patch}}) - \text{nearest}_M\|_2$

    이미지 단위 score: 모든 patch score 중 최대값

    추가로 PatchCore는 점수가 가장 큰 patch에 대해 re-weighting(주변 k개 NN의 평균 거리로 정규화) — 이상 patch가 정말 외톨이인지 확인.

4. 수식 (가능한 한 간단히)

Coreset 선택

$$ m_i = \arg\max_{m \in P \setminus M_{i-1}} \min_{m' \in M_{i-1}} \|m - m'\|_2 $$

매 iteration마다, 이미 뽑힌 것들로부터 가장 멀리 있는 vector를 추가.

Anomaly Score

테스트 patch feature $f^*$에 대해:

$$ s(f^*) = \min_{m \in M} \|f^* - m\|_2 $$

이미지 score:

$$ S(x) = \max_{f^* \in \text{patches}(x)} s(f^*) $$

(여기까지가 핵심. re-weighting은 안정성 보강용 트릭.)

5. 주요 설계 결정과 그 이유

결정왜?
Pretrained 사용, 학습 없음학습 데이터 부족한 산업 환경에 유리. ImageNet feature가 일반화됨
layer 2+3 concatmid-level feature가 산업 결함(긁힘, 얼룩 등) 검출에 최적
coreset 1% 유지정확도 손실 거의 없으면서 memory ~100배 감소
최대값 pool (이미지 score)한 patch만 매우 이상해도 전체 이미지 이상 — 산업 검사 철학과 일치

6. 산업 현장 관점 — 장단점

✅ 강점

  • 학습 거의 불필요 — feature extraction은 정상 이미지 1회 forward로 끝
  • MVTec AD 99.2% AUROC — anomalib v2.4에 그대로 포함
  • 재현성 매우 높음 — Amazon 공식 구현 + anomalib 독립 재현
  • 라이선스 Apache 2.0 → 상업 사용 가능

⚠ 한계

  • 추론 latency가 모델 크기 의존 — memory bank 클수록 느림
  • 새 도메인 약점 — X-ray, 적외선 등에선 ImageNet feature 안 통할 수 있음
  • 비교 가능성은 별개 — Phase 4에서 발견한 bootstrap CI 문제는 PatchCore 포함 모든 모델에 적용

8. 코드 — anomalib 실제 호출 (중복 구현 불필요)

from anomalib.models import Patchcore
from anomalib.data import MVTec
from anomalib.engine import Engine

datamodule = MVTec(category="bottle")
model = Patchcore(
    backbone="wide_resnet50_2",
    layers=["layer2", "layer3"],
    coreset_sampling_ratio=0.1,   # 10% (논문 기본은 1%)
    num_neighbors=9,
)

engine = Engine()
engine.fit(model=model, datamodule=datamodule)
results = engine.test(model=model, datamodule=datamodule)
중복 구현 방지 원칙 이게 전부입니다. 우리는 직접 짤 필요 없음.

9. 손으로 따라가보는 미니 구현 (이해 확인용)

만약 anomalib 없이 PatchCore의 영혼만 30줄로 쓴다면:

import torch, torchvision

# 1. Pretrained feature extractor
backbone = torchvision.models.wide_resnet50_2(weights="IMAGENET1K_V1").eval()

def extract(x):                    # x: (B, 3, 224, 224)
    feats = []
    h = x
    for name, layer in backbone._modules.items():
        h = layer(h)
        if name == "layer2":        # (B, 512, 28, 28)
            feats.append(h)
        if name == "layer3":        # (B, 1024, 14, 14) → upsample to 28
            feats.append(torch.nn.functional.interpolate(h, size=(28, 28)))
            break
    return torch.cat(feats, dim=1)  # (B, 1536, 28, 28)

# 2. Memory bank from normal images
@torch.no_grad()
def build_bank(normal_loader):
    bank = []
    for x in normal_loader:
        f = extract(x)                      # (B, 1536, 28, 28)
        f = f.permute(0, 2, 3, 1).reshape(-1, 1536)  # (B*784, 1536)
        bank.append(f)
    return torch.cat(bank, dim=0)           # (N, 1536)

# 3. (coreset 생략 — 핵심 아이디어 이해엔 불필요)

# 4. Score test image
@torch.no_grad()
def score(test_image, bank):
    f = extract(test_image).permute(0, 2, 3, 1).reshape(-1, 1536)
    dists = torch.cdist(f, bank).min(dim=1).values
    return dists.max().item()

30줄로 핵심을 보여주지만, 실전에서는 anomalib v2.4 구현(coreset, FAISS 가속, OpenVINO export 등)을 그대로 쓰는 것이 합리적.
