1. 왜 EfficientAD가 필요했는가

2022~2023년의 PatchCore·CFLOW·FastFlow 등은 모두 99%대 AUROC에 도달했으나, 산업 적용에는 두 가지 부족함이 남았습니다.

부족함이유
속도WideResNet-50 backbone · memory bank 검색은 GPU에서도 수십 ms — production line 사이클타임(< 1초)에서 검사 1건만 이걸 쓰기엔 비효율
"논리적 이상" 검출 약함"긁힘" 같은 구조적(structural) 이상은 잘 잡지만, "부품이 빠짐 / 잘못된 위치에 있음" 같은 논리적(logical) 이상은 patch만으로 검출하기 어려움
EfficientAD의 두 가지 답
  1. 경량 backbone(PDN) + Hard feature loss → latency 2 ms 달성
  2. Autoencoder 분기 추가 → 전체 이미지 맥락(논리적 이상) 검출

핵심 비유
  • Teacher: "정상 세계관"을 미리 다 알고 있는 전문가 (학습 X, 고정)
  • Student: 정상만 보며 Teacher 흉내내기를 배움. 정상은 잘 흉내, 이상은 흉내 못 냄 (배운 적 없으니까)
  • Autoencoder: 정상 이미지를 "전역적으로" 보고 Teacher 출력을 재구성. 부품 위치 같은 구조 정보를 압축해 저장

3. 알고리즘 (4단계)

  1. PDN — Patch Description Network

    EfficientAD는 ResNet 대신 4-layer 경량 CNN을 직접 설계해 사용합니다.

    Layer출력 shape (256 입력 가정)역할
    Conv 4×4, stride 1(128, 253, 253)저수준 텍스처
    Conv 4×4, stride 2(256, 125, 125)중수준
    Conv 3×3, stride 1(256, 122, 122)—
    Conv 4×4, stride 1(384, 56, 56)최종 patch feature

    PDN의 receptive field는 약 33×33 — 충분히 큰 영역을 보면서도 ResNet 대비 연산량이 ~30배 적음. PDN-S(Small)와 PDN-M(Medium)이 있고, S가 가장 빠름.

    Teacher PDN은 ImageNet 전체에 대해 사전학습 (논문에서 distillation으로 ImageNet 정보 압축). 다운로드 가능한 weight 제공.

  2. Student–Teacher Distillation

    정상 이미지 데이터셋에서 Student PDN이 Teacher PDN의 출력을 그대로 흉내내도록 학습:

    $$ \mathcal{L}_{\text{ST}} = \frac{1}{C \cdot H \cdot W} \sum_{c,i,j} \big( T(x)_{c,i,j} - S(x)_{c,i,j} \big)^2 $$

    $T$는 Teacher, $S$는 Student. 정상 이미지만으로 학습하므로 Student는 정상 영역만 흉내낼 줄 알게 됨. 이상 영역이 들어오면 Student가 Teacher를 못 따라가서 차이가 큼.

  3. Hard Feature Loss — Top-K 정예만 학습

    모든 patch에 대해 MSE를 계산하면 학생이 "너무 잘" 흉내내서 이상도 잘 흉내내기 시작함 (over-distillation). EfficientAD는 가장 차이가 큰 상위 0.1% patch만 gradient에 사용:

    $$ \mathcal{L}_{\text{hard}} = \frac{1}{|H|} \sum_{(c,i,j) \in H} \big( T - S \big)^2_{c,i,j}, \quad H = \text{Top-K}_{p=0.001} $$

    효과 2가지:

    • Regularization — Student가 "너무 잘 흉내"내는 것을 방지 → 이상 일반화 능력 유지
    • 학습 효율 — 쉬운 patch는 무시 → 수렴 빠름
    주의 hard mining 비율(0.1%)이 EfficientAD의 가장 민감한 hyperparameter. 너무 낮으면 학습 신호 부족, 너무 높으면 over-fit.

  4. Autoencoder 분기 — Logical Anomaly

    Student–Teacher만으로는 "부품이 잘못된 위치" 같은 전역 구조 이상은 검출 어려움. EfficientAD는 추가 Autoencoder를 두어 Teacher feature를 압축·복원:

    $$ \mathcal{L}_{\text{AE}} = \frac{1}{C \cdot H \cdot W} \sum \big( T(x) - A(x) \big)^2 $$

    $A$는 Autoencoder. AE는 정상 분포의 "전역적 평균"을 학습 — 이상은 평균에서 멀리 떨어져 reconstruction 실패.

    그리고 Student는 두 번째 출력 채널도 가져서, 이 채널은 AE의 출력을 흉내내도록 학습:

    $$ \mathcal{L}_{\text{STAE}} = \frac{1}{C \cdot H \cdot W} \sum \big( A(x) - S'(x) \big)^2 $$

    $S'$는 Student의 두 번째 출력. Student–AE 차이가 크면 이상 (Student는 AE의 정상 분포 추정을 따라가야 하는데 못 따라가면 이상 신호).

4. 추론 — 두 score 결합

테스트 이미지 $x$가 들어오면 다음 두 map을 계산:

Map수식의미
Local (구조적)$M_{\text{local}} = \|T(x) - S(x)\|^2$Student가 Teacher를 못 따라간 정도 — 작은 결함
Global (논리적)$M_{\text{global}} = \|A(x) - S'(x)\|^2$Student-AE 차이 — 전역 이상

두 map을 quantile-normalize한 뒤 평균:

$$ M(x) = \tfrac{1}{2}\big( \text{norm}(M_{\text{local}}) + \text{norm}(M_{\text{global}}) \big) $$

이미지 score = $M(x)$의 최대값. Pixel-level score map은 $M(x)$ 그대로.

Quantile Normalization 정상 validation 데이터에서 미리 측정한 $M_{\text{local}}$, $M_{\text{global}}$의 분포 quantile로 정규화. 두 분포의 scale 차이를 없애 평등 결합.

6. 산업 현장 관점 — 장단점

✅ 강점

  • 2 ms/img (RTX A6000, S) — production line latency 압도적 충족
  • MVTec 99.6%(S) / 99.8%(M)
  • 논리적 이상도 일부 검출 — 다른 KD 모델 대비 차별점
  • anomalib 통합, Apache 2.0
  • edge 디바이스 적용 사례 다수

⚠ 한계

  • Hyperparameter 민감 — top-K 비율, 학습 epoch, AE 차원 등 튜닝 필요
  • Teacher weight 외부 의존 — 논문 저자 제공 weight 다운로드 필요
  • 3개 네트워크 동시 운영 — Memory bank 없는 대신 학습 시간·메모리 ↑
  • VisA 등에서 PatchCore 대비 우위가 일관되지 않음 (98.1% 보고)

9. 손으로 따라가보는 미니 구현 (이해 확인용)

EfficientAD의 영혼을 60줄 안으로:

import torch
import torch.nn as nn
import torch.nn.functional as F

# --- 1. PDN (간소화: 4-conv) ---
class PDN(nn.Module):
    def __init__(self, out_channels=384):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(3,   128, 4, stride=1), nn.ReLU(),
            nn.AvgPool2d(2, 2),
            nn.Conv2d(128, 256, 4, stride=1), nn.ReLU(),
            nn.Conv2d(256, 256, 3, stride=1), nn.ReLU(),
            nn.Conv2d(256, out_channels, 4, stride=1),
        )
    def forward(self, x):
        return self.net(x)

# Teacher: pretrained weight 로드 (논문 제공)
teacher = PDN().eval()
# teacher.load_state_dict(torch.load("teacher.pth"))   # 실전에선 필수

# Student: 두 채널 — 첫 384는 teacher 흉내, 다음 384는 AE 흉내
student = PDN(out_channels=384*2).train()

# Autoencoder: 입력 이미지 → teacher feature 재구성
autoencoder = nn.Sequential(
    # encoder
    nn.Conv2d(3,32,4,2,1), nn.ReLU(),
    nn.Conv2d(32,64,4,2,1), nn.ReLU(),
    # decoder (간소화)
    nn.ConvTranspose2d(64,384,4,2,1), nn.ReLU(),
    nn.ConvTranspose2d(384,384,4,2,1),
).train()

# --- 2. Training step ---
def train_step(x, teacher, student, autoencoder, opt):
    with torch.no_grad():
        t = teacher(x)                  # (B, 384, h, w)
    s = student(x)                       # (B, 768, h, w)
    s_t, s_ae = s[:, :384], s[:, 384:]
    a = autoencoder(x)                   # (B, 384, h, w)

    # Hard feature loss (top 0.1%)
    err = (t - s_t) ** 2                 # (B, 384, h, w)
    k = int(err.numel() * 0.001)
    L_hard = torch.topk(err.flatten(), k).values.mean()

    L_ae   = ((t - a) ** 2).mean()
    L_stae = ((a.detach() - s_ae) ** 2).mean()

    loss = L_hard + L_ae + L_stae
    opt.zero_grad(); loss.backward(); opt.step()
    return loss.item()

# --- 3. Inference ---
@torch.no_grad()
def score(x, teacher, student, autoencoder):
    t = teacher(x); s = student(x); a = autoencoder(x)
    s_t, s_ae = s[:, :384], s[:, 384:]
    m_local  = ((t - s_t)  ** 2).mean(dim=1)   # (B, h, w)
    m_global = ((a - s_ae) ** 2).mean(dim=1)
    m = 0.5 * (m_local + m_global)             # quantile-normalize 생략
    return m.amax(dim=(1, 2))                   # image score

실전에서는 quantile normalization, padding 처리, Teacher pretrained weight 로딩, 적절한 학습률 스케줄 등이 추가됨. anomalib v2.4의 구현은 이를 모두 처리.

10. PatchCore와의 결합 — 본 연구 Pareto Frontier

본 연구 P1(IF-PROTO)에서 두 모델은 latency–recall 평면의 양 극단을 점유:

recall@FPR=0.1%
    ▲
0.6 │           ● PatchCore (memory bank 1%)
    │
0.5 │     ●─── EfficientAD-M
    │
0.4 │   ● EfficientAD-S
    │
0.3 │
    └──────────────────────────►
      2ms   10ms     30ms    100ms+   latency

실측이 가설을 확인하면, 다음과 같은 결론 후보가 도출:

  • "recall이 latency를 희생하지 않고 얻어진다" — 두 모델이 비슷한 recall이면
  • "FPR 임계 영역에서 빠른 모델이 무너진다" — EfficientAD recall이 PatchCore보다 한참 낮으면
  • "개별 모델보다 ensemble이 우월" — 두 모델 score stacking이 단일 대비 유의한 개선이면

이것이 Phase 4 단계에서 실측해야 할 핵심 가설입니다.
