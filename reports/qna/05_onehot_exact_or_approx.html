One-hot에서 CE = KL은 "완전히 같다"인가 "근사"인가?

질문자: 연구자 (사용자) 작성일: 2026-05-27 관련: 개념 ② · 개념 ③

0. 한 줄 답

결론 — 둘 다 일리가 있음
  • 표준 ML 수학 (convention $0 \log 0 := 0$) 하에서는 완전히 동일 (exact equality)
  • 엄격한 함수 해석학 관점에서는 $0 \log 0$이 부정형이라 "정의 자체가 안 되고, 극한으로만 0에 수렴" → 사용자의 "근사" 직관이 정확
표준 ML 라이브러리(PyTorch, TF, JAX 등)와 정보이론 교과서는 모두 convention을 적용해 exact로 다룹니다. 그러나 사용자의 의심은 수학적으로 정당한 회의이며, 본 문서가 그 근거를 정리합니다.

1. 사용자의 의구심을 분해

One-hot $p = (1, 0, 0, 0)$에 대해 entropy를 직접 계산해보면:

$$ H(p) = -\sum_{i=1}^{4} p_i \log p_i = -1 \cdot \log 1 \;-\; 0 \cdot \log 0 \;-\; 0 \cdot \log 0 \;-\; 0 \cdot \log 0 $$

여기서 두 종류의 항이 등장합니다:

항값문제
$-1 \cdot \log 1$ $-1 \cdot 0 = 0$ 없음 ($\log 1 = 0$, 곱은 0)
$-0 \cdot \log 0$ $-0 \cdot (-\infty)$ 부정형 — $0 \times \infty$은 수학적으로 미결정
핵심 $0 \cdot (-\infty)$는 수학적 부정형 (indeterminate form)입니다. $0/0$, $\infty - \infty$와 같이 "고유한 답이 없는 식". 그래서 "$H(p) = 0$이다"라고 직접 말할 수 없고, 추가 규약이 필요합니다.

사용자의 의구심은 이 부정형을 직시한 것입니다. 정당한 의심.

2. 두 가지 입장

2.1 Convention A — 표준 ML / 정보이론

$\boxed{\; 0 \log 0 := 0 \;}$ — 정의로 도입합니다.

근거는 극한:

$$ \lim_{x \to 0^+} x \log x = 0 $$

증명 (L'Hôpital 또는 직접):

$$ \lim_{x \to 0^+} x \log x = \lim_{x \to 0^+} \frac{\log x}{1/x} \stackrel{\text{L'Hôpital}}{=} \lim_{x \to 0^+} \frac{1/x}{-1/x^2} = \lim_{x \to 0^+} -x = 0 $$

그래서 $f(x) = x \log x$ 함수를 $x = 0$에서 연속적으로 확장하여 $f(0) := 0$으로 정의합니다 (continuous extension).

Convention A의 입장 "$0 \log 0$은 정의되지 않는 게 아니라, 연속 확장에 의해 정확히 0이다." 이 규약 하에서 $H(p)$, $D_{KL}$, $H(p,q)$가 모두 잘 정의되고, one-hot의 경우 $H(p) = 0$ 정확히.

2.2 Convention B — 엄격한 함수 해석학

$x \log x$는 정의역이 $x > 0$ ($\log$가 정의 안 되는 점 제외). $x = 0$에서 함수 정의 자체가 안 됨. 따라서:

  • One-hot $p$에 대한 $H(p)$는 잘 정의되지 않음
  • 다만 "$p$가 one-hot에 한없이 가까워질수록 $H(p) \to 0$" 라는 극한 성질만 성립
Convention B의 입장 "엄밀히 말하면 one-hot에서 $H(p)$는 정의 안 됨. $H(p,q)$와 $D_{KL}$이 같은지 묻는 것 자체가 부적절." 그러나 이 입장으로는 ML 실무가 진행 안 됨 → 결국 Convention A로 돌아옴.

2.3 어느 쪽이 옳은가?

수학적으로는 둘 다 정합적입니다. 어떤 입장을 채택하느냐는 분야 관습:

  • 정보이론, 머신러닝, 통계학: Convention A 보편 사용. 모든 교과서·라이브러리
  • 엄격한 해석학 / 실해석 강의: Convention B를 명시적으로 다루고, "연속 확장"을 별도 단계로 분리

본 문서 나머지는 Convention A 기준으로 진행합니다 (ML 실무 관행).

3. Convention A로 정확한 계산 — One-hot 수치 예시

$p = (1, 0, 0, 0)$ (one-hot, 정답 클래스 인덱스 $y = 1$), $q = (0.4, 0.3, 0.2, 0.1)$로 모든 양을 정확히 계산해봅니다 (자연로그 nats).

중요한 차이 $D_{KL}$ 계산에서 $i \neq 1$ 항들은 convention이 필요합니다. 이유: $\log(0/q_i) = \log 0 = -\infty$. 따라서 $0 \cdot (-\infty)$ 부정형 등장. Convention A로 0으로 정의.

즉 $D_{KL}$의 "exact equality"는 convention에 의존합니다. 사용자의 의심이 향한 정확한 지점.

3.4 항등식 검증

$$ H(p, q) = D_{KL}(p \| q) + H(p) $$

$$ 0.9163 \stackrel{?}{=} 0.9163 + 0 $$

$$ 0.9163 = 0.9163 \quad \checkmark $$

Convention A 하에서 정확히 일치 — exact.

4. 극한 분석 — Convention B 관점의 검증

사용자의 직관 ("근사")이 어디서 오는지 보려면, one-hot을 정확한 점이 아니라 한없이 one-hot에 가까운 분포의 극한으로 보면 됩니다.

4.1 $\epsilon$-perturbation 분포

$$ p_\epsilon = \Big( 1-\epsilon, \; \tfrac{\epsilon}{3}, \; \tfrac{\epsilon}{3}, \; \tfrac{\epsilon}{3} \Big) $$

$\epsilon > 0$인 한 모든 원소가 양수 → 어떤 convention도 필요 없는 well-defined 분포. $\epsilon \to 0$일 때 one-hot으로 수렴.

4.2 각 양의 극한 계산

$H(p_\epsilon)$

$$ H(p_\epsilon) = -(1-\epsilon)\ln(1-\epsilon) \;-\; 3 \cdot \tfrac{\epsilon}{3} \ln \tfrac{\epsilon}{3} $$

$$ = -(1-\epsilon)\ln(1-\epsilon) \;-\; \epsilon \ln \tfrac{\epsilon}{3} $$

$\epsilon \to 0$일 때:

  • $(1-\epsilon)\ln(1-\epsilon) \to 1 \cdot 0 = 0$
  • $\epsilon \ln(\epsilon/3) = \epsilon \ln \epsilon - \epsilon \ln 3 \to 0 - 0 = 0$ (왜냐하면 $\epsilon \ln \epsilon \to 0$)

따라서 $\lim_{\epsilon \to 0^+} H(p_\epsilon) = 0$.

$H(p_\epsilon, q)$ (with $q = (0.4, 0.3, 0.2, 0.1)$)

$$ H(p_\epsilon, q) = -(1-\epsilon)\ln 0.4 \;-\; \tfrac{\epsilon}{3}(\ln 0.3 + \ln 0.2 + \ln 0.1) $$

$\epsilon \to 0$일 때:

$$ \lim_{\epsilon \to 0^+} H(p_\epsilon, q) = -1 \cdot \ln 0.4 - 0 = -\ln 0.4 = 0.9163 $$

$D_{KL}(p_\epsilon \| q)$

$$ D_{KL}(p_\epsilon \| q) = (1-\epsilon) \ln \frac{1-\epsilon}{0.4} + 3 \cdot \frac{\epsilon}{3} \ln \frac{\epsilon/3}{q_j} $$

여기서 $q_j \in \{0.3, 0.2, 0.1\}$ — 합산으로 풀어 쓰면 다소 복잡하지만 핵심은:

$\epsilon \to 0$일 때 두 번째 합이 $\epsilon \ln \epsilon$ 형태로 0으로 수렴.

$$ \lim_{\epsilon \to 0^+} D_{KL}(p_\epsilon \| q) = 1 \cdot \ln(1/0.4) = \ln 2.5 = 0.9163 $$

사용자의 "근사" 직관은 $\epsilon > 0$인 모든 perturbation에서 정확히 성립. 그러나 정확히 one-hot ($\epsilon = 0$)이 되면 convention에 의해 exact가 됨.

6. PyTorch에서의 실제 동작

이론적 정밀성과 실제 코드의 차이를 봅니다.

6.1 PyTorch의 처리

import torch
import torch.nn.functional as F

# One-hot target
p = torch.tensor([1.0, 0.0, 0.0, 0.0])
q = torch.tensor([0.4, 0.3, 0.2, 0.1])

# === Cross-Entropy (분포 입력 형태) ===
# 직접 계산: -Σ p_i log(q_i)
ce = -(p * torch.log(q)).sum()
print(f"CE direct: {ce.item():.4f}")   # 0.9163

# === KL Divergence (PyTorch built-in) ===
# F.kl_div(input=log_q, target=p, reduction='sum')
# 내부에서 target * (log_target - input)을 계산하고, target=0인 항은 0으로 처리
kl = F.kl_div(torch.log(q), p, reduction='sum')
print(f"KL (F.kl_div): {kl.item():.4f}")  # 0.9163

# === Entropy of one-hot ===
# 직접 계산하면 NaN! (0 * log(0) = 0 * -inf = NaN in IEEE 754)
H_naive = -(p * torch.log(p)).sum()
print(f"H(p) naive: {H_naive.item()}")  # nan

# 올바른 계산: 0*log(0) 항을 안전하게 처리
mask = p > 0
H_safe = -(p[mask] * torch.log(p[mask])).sum()
print(f"H(p) safe:  {H_safe.item():.4f}")  # 0.0000

6.2 핵심 관찰

  • $H(p, q)$의 계산: $0 \cdot \log q_i$만 등장 ($q_i > 0$). NaN 발생 안 함. 안전.
  • $H(p)$의 직접 계산: $0 \cdot \log 0 = 0 \cdot (-\infty)$ → IEEE 754에서 NaN. 실제로 코드가 깨짐.
  • 그래서 라이브러리는 masking 또는 epsilon-smoothing으로 안전하게 처리.
  • $D_{KL}$도 마찬가지: 라이브러리가 target=0 항을 명시적으로 0으로 두어 NaN 회피.
실무적 함정 이론적으로 convention $0 \log 0 = 0$이지만 floating-point에서는 NaN. 학습 loss가 NaN으로 나오는 흔한 원인. One-hot target에 KL을 직접 적용하지 말고, F.cross_entropy를 쓰거나 masking으로 안전 처리.

7. 표 — 6가지 입장의 비교

입장 One-hot $H(p)$ $H(p,q)$ vs $D_{KL}$ 본 연구의 관점
수학적 (Convention A: $0\log 0 = 0$) $= 0$ exact 완전히 동일 (exact) 표준
수학적 (Convention B: 부정형) 정의 안 됨 비교 불가 거의 사용 안 함
극한 관점 $\lim_{p \to \text{one-hot}} H(p) = 0$ $\epsilon$이 작을수록 근사, 극한에서 일치 "근사" 직관의 정확한 형식화

8. 결론 — 사용자의 직관에 대한 평가

정리 사용자의 "근사가 정답일 것 같다"는 직관은 다음 두 의미에서 정확한 수학적 통찰입니다:
  1. $0 \cdot \log 0$가 부정형이라는 점을 직시 — 일반 수학에서 자명하지 않은 부분
  2. $\epsilon$-perturbation의 모든 양수 값에서 정확히 성립 — 두 양이 완전 같지 않고 차이가 $\epsilon \log \epsilon$ 정도
그러나 표준 ML 수학에서는 convention $0 \log 0 = 0$을 적용해 exact equality로 사용. 이 convention은:
  • 연속 확장(continuous extension)으로 정당화됨
  • 모든 정보이론 교과서, ML 라이브러리에서 채택
  • $\epsilon \to 0$ 극한의 값으로 정의된 것이므로 "근사"의 형식화이기도 함
즉 두 입장은 같은 사실을 다른 언어로 표현합니다. 사용자의 의심은 정당하고, 표준 결론도 정당합니다.

9. 이전 보고서의 정정

개념 ② 핵심 통찰 4번 "One-hot에서 $H(p) = 0$이므로 CE = KL exactly" 표현은 convention A 적용 시라는 단서가 누락되어 있었습니다. 정확한 표현:

정정된 표현 "One-hot target에 대해, 정보이론의 표준 규약 $0 \log 0 := 0$을 적용하면 $H(p) = 0$이 되고, 결과적으로 $H(p, q) = D_{KL}(p \| q)$가 convention 하에서 정확히 일치. $\epsilon > 0$인 어떤 perturbation에 대해서도 두 양은 $\epsilon$에 비례하는 차이를 가지며, $\epsilon \to 0$ 극한에서 수렴."

이는 결론을 바꾸지 않습니다 — ML 실무에서는 여전히 "동일"로 다루는 것이 표준. 다만 그 동일성이 convention에 의존한다는 점을 명시.
