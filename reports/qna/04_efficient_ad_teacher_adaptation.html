분석 보고 · Q&A ④

EfficientAD의 Teacher 모델도 ImageNet 외에 임의의 추가 학습을 시키는 건 부적절한가?

질문자: 연구자 (사용자) 작성일: 2026-05-27 관련: 분석 ① 백본 적응 · 분석 ③ PDN 명확화

0. 한 줄 답

결론 (한 줄) PatchCore의 경우보다 더 부적절합니다. EfficientAD의 Teacher는 단순한 feature extractor가 아니라 Student와 짝지어진 reference이기 때문에, Teacher를 목표 도메인에 적응시키면 score가 이중으로 무너집니다 (compound collapse).

1. 왜 EfficientAD의 Teacher는 PatchCore의 backbone보다 더 민감한가

PatchCore와 EfficientAD의 score 공식을 비교해보면 차이가 분명합니다.

PatchCore

$$ S_{\text{PC}}(x) = \min_{m \in M} \|f_\theta(x) - m\|_2 $$

여기서 $f_\theta$는 backbone, $M$은 정상 feature 메모리 뱅크. Backbone은 score 계산에 1번만 등장.

EfficientAD

$$ S_{\text{EAD}}(x) = \|T(x) - S(x)\|^2 $$

여기서 $T$는 Teacher, $S$는 Student. Teacher는 score의 한 항(term)으로 직접 등장하고, 동시에 Student의 학습 target이기도 함:

$$ S^* = \arg\min_\phi \sum_{x \in \text{normal}} \|T(x) - S_\phi(x)\|^2 $$

치명적 차이 PatchCore에서 backbone이 무너지면 → 정상의 거리가 작아지고 이상의 거리도 따라 작아짐 (1중 collapse).
EfficientAD에서 Teacher가 무너지면 → (1) Teacher 자체가 정상에 특화 + (2) Student가 그 특화된 Teacher를 더 잘 흉내냄 → score의 두 항이 동시에 작아짐 (2중 collapse).

2. 수학으로 본 compound collapse

Teacher $T_\psi$를 target domain의 정상 이미지로 추가 학습한다고 가정. $\psi$가 정상 분포 $\mathcal{D}_{\text{normal}}$에 더 잘 fit 되면:

  1. $T_\psi(x_{\text{normal}})$가 좁은 매니폴드로 모임 (feature collapse)
  2. Student는 그 좁은 target을 더 쉽게 흉내 — Hard Feature Loss(top 0.1%)도 의미가 약해짐
  3. $T_\psi(x_{\text{anomaly}})$도 학습되지 않은 입력이라 무작위 방향을 갖지만, 정상과의 차이를 만드는 "다양한 의미 채널"이 사라진 상태
  4. 결과적으로 $\|T_\psi(x_{\text{anomaly}}) - S_\phi(x_{\text{anomaly}})\|$이 정상의 경우와 구분되지 않음

수식으로 표현하면 Teacher가 fit될수록

$$ \mathbb{E}_{\text{normal}}[\|T_\psi - S_\phi\|^2] \to 0 \quad \text{(원래 의도)} $$

$$ \mathbb{E}_{\text{anomaly}}[\|T_\psi - S_\phi\|^2] \to 0 \quad \text{(의도하지 않은 부작용)} $$

두 값의 격차 (anomaly score gap)가 모두 0으로 수렴 → 검출력 소실.

PatchCore와 본질적 차이 PatchCore는 단일 collapse라 회피 가능한 영역이 있다 (e.g., 메모리 뱅크의 다양성이 일부 보존되면 일부 anomaly는 여전히 멀리 떨어짐). EfficientAD는 Teacher와 Student가 학습 시 직접 결합되어 있어 Teacher의 collapse가 Student의 over-distillation으로 즉시 전파됨.

3. 4갈래 분류 — EfficientAD 버전

분석 ①에서 PatchCore에 적용한 4갈래 분류를 EfficientAD에 다시 적용해보면, 결론이 더 강하게 부정적입니다.

방식구체적EfficientAD에서의 위험도PatchCore와 비교
(가) Teacher를 정상 데이터로 supervised fine-tune 기존 Teacher PDN weight 로드 후 정상 이미지로 reconstruction/classification 추가 학습 매우 고위험 — compound collapse, 거의 확실히 악화 PatchCore 케이스보다 위험 ↑↑
(나) Teacher를 self-supervised로 target에 적응 DINO/MoCo로 target 이미지에 self-supervised 학습 고위험 — Teacher가 target에 특화되는 순간 distillation 균형 깨짐 PatchCore 케이스보다 위험 ↑
(다) 큰 source 네트워크부터 다시 distillation "target+ImageNet" 데이터로 큰 모델 (WRN-101 등) 학습 → 다시 PDN으로 distill 중위험 — 정확도 가능, 비용 ↑↑ PatchCore에는 없는 옵션 (Teacher가 distilled라 가능)
(라) Teacher를 다른 distilled 버전으로 교체 DINOv2 등에서 distill한 PDN-style teacher로 교체 (학습 안 함) 저위험 — 가장 안전한 대안 PatchCore의 "DINOv2 backbone 교체"에 대응
(마) Teacher는 그대로, Student/AE 학습만 강화 대상 정상 이미지로 Student와 AE를 더 오래/더 강하게 학습 안전 — 원래 의도된 적응 경로 PatchCore에는 대응 없음 (학습 가능 모듈이 없음)

4. PatchCore vs EfficientAD — 위험도 직접 비교

측면PatchCore backbone fine-tuneEfficientAD Teacher fine-tune
Score에 등장 횟수 1번 ($\|f - m\|$) 2번 (term + Student 학습 target)
다른 컴포넌트와의 결합 독립적 (memory bank는 backbone 출력의 함수일 뿐) Student와 distillation으로 결합
Feature collapse 시 효과 정상 거리 ↓, 이상 거리도 ↓ (1중) 정상 mismatch ↓, 이상 mismatch도 ↓ (2중)
회복 가능성 memory bank 일부 다양성 보존 시 회복 가능 Student도 함께 무너져 회복 어려움
Hard Feature Loss의 보호 효과 (해당 없음) Teacher가 collapse하면 top-K도 의미 약화
학계 보고 PatchCore 원논문 §4.4에서 "fine-tune 모두 악화" 명시 EfficientAD 원논문도 Teacher frozen을 강조. 변형 시도 보고 극소수
한눈에 PatchCore의 backbone fine-tune은 "비추천이지만 일부 도메인에선 효과 가능"이지만, EfficientAD의 Teacher fine-tune은 "거의 모든 경우에 비추천"입니다.

5. EfficientAD의 정상적인 도메인 적응 경로

그렇다면 대상 도메인에 EfficientAD를 적응시키려면 어떻게 해야 할까요? 다행히 EfficientAD는 학습 가능한 모듈을 이미 두 개 갖고 있어, PatchCore보다 적응 경로가 풍부합니다.

5.1 우선순위 1 — Student와 Autoencoder 학습을 더 풍부하게

방법구체적예상 효과
학습 데이터 증대 정상 이미지 더 수집 (가능하면 다양한 조명·각도·batch) 가장 큰 ROI. anomalib 기본 epoch도 충분히 늘려보기
해상도 ↑ 입력을 256 → 512 → 1024로 (PDN의 receptive field가 33×33이라 큰 입력에서 더 미세한 결함 검출) small-defect recall ↑, latency ↑
AE bottleneck 차원 조정 대상 도메인의 "전역 변동성"에 맞게 조정 (단순 제품이면 더 작게, 복잡하면 더 크게) logical anomaly 검출력 조절
Hard Feature Loss 비율 조정 기본 0.1%에서 시작. 학습이 너무 빠르면 (over-distill 의심) 0.05%로, 학습이 느리면 0.3%로 학습-검출 균형

5.2 우선순위 2 — Teacher를 "교체"하기 (학습 X, 다른 distilled 버전 사용)

Teacher PDN을 학습시키는 게 아니라 아예 다른 backbone으로 교체하는 것은 안전한 옵션입니다.

교체 후보출처구현 비용
DINOv2-distilled PDN DINOv2 ViT → PDN으로 distill (사전에 한 번만) 중 (distillation 학습 1회)
다른 ImageNet 모델 distill ConvNeXt, Swin 등에서 PDN으로 distill 중
Domain-mixed pretraining + distill "target + ImageNet" 데이터로 큰 모델 pretrain → PDN distill 고 (pretrain + distill 2단계)

이 경우 Teacher는 여전히 학습되지 않습니다 — 새로운 distillation으로 만든 weight를 로드하기만 합니다. 이는 분석 ①의 (라) "DINOv2 백본 교체"에 정확히 대응.

5.3 우선순위 3 — 새로 distillation pipeline 구축 (대형 작업)

특수 도메인 (X-ray, IR 등 ImageNet 격차가 클 때):

1. 대상 도메인 + 일반 ImageNet 데이터를 합쳐 큰 모델(WRN-101 등)을 self-supervised pretrain
2. 그 모델에서 PDN으로 distill (EfficientAD 원래 절차와 동일)
3. 만들어진 Teacher PDN을 EfficientAD에 로드 (학습 X)
4. Student + AE만 정상 데이터로 학습

이 경로는 (가) Teacher fine-tune의 모든 위험을 회피하면서 동시에 도메인 특화 효과를 얻을 수 있는 유일한 안전한 방법입니다. 단, distillation pipeline 구축 비용 (수일 작업).

8. 결론 (한 줄 — 다시)

최종 답 EfficientAD의 Teacher를 임의로 추가 학습하는 것은 PatchCore의 backbone fine-tune보다 더 위험합니다. 이유: Teacher는 score 공식의 한 항이자 Student의 학습 target이라, fine-tune 시 score의 두 항이 동시에 collapse(compound collapse). 대안: (1) Student와 AE 학습을 강화, (2) Teacher를 다른 distilled 버전으로 교체, (3) 큰 source부터 다시 distillation. Teacher를 직접 학습하는 (가) 옵션은 검증 실패 사례로 보고하는 것이 학술적 기여가 됨.
