EfficientAD에서 PDN은 Teacher와 Student에 동일한 네트워크인가?

질문자: 연구자 (사용자) 작성일: 2026-05-27 관련: 학습 ② EfficientAD

0. 한 줄 답

결론 PDN 아키텍처는 Teacher와 Student가 같지만, 3가지 차이가 있습니다:
  1. 출력 채널 수: Student는 Teacher의 2배 (384 → 768)
  2. 학습 방식: Teacher는 ImageNet 사전학습 후 frozen, Student는 정상 데이터로 처음부터 학습
  3. 가중치 초기화: Teacher는 미리 받은 weight 로드, Student는 random
그리고 Autoencoder는 PDN이 아닙니다 — 완전 별도의 encoder-decoder 구조.

1. PDN 아키텍처 — Teacher / Student 공통

EfficientAD는 두 가지 PDN 변형을 정의합니다. PDN-S (Small)과 PDN-M (Medium). Teacher와 Student는 같은 변형의 아키텍처를 공유합니다.

PDN-S 구조 (Teacher 기준 출력 채널 384)

Layer커널 · stride출력 채널비고
Conv 14×4, stride 1128입력 3
AvgPool 12×2—다운샘플
Conv 24×4, stride 1256—
AvgPool 22×2—다운샘플
Conv 33×3, stride 1256—
Conv 44×4, stride 1384 (Teacher) / 768 (Student)최종 출력

한 줄 요약: Conv 1~3은 Teacher와 Student가 완전히 같고, 마지막 Conv 4의 출력 채널 수만 다릅니다.

2. 왜 Student는 출력 채널이 2배인가?

Student는 두 가지를 동시에 흉내내야 합니다.

출력 채널 분할역할대응 loss
채널 1~384 ($S$) Teacher 흉내 — 정상 patch에서 Teacher feature와 같아지도록 $\mathcal{L}_{\text{ST}} = \|T(x) - S(x)\|^2$
채널 385~768 ($S'$) Autoencoder 흉내 — 정상 patch에서 AE 출력과 같아지도록 $\mathcal{L}_{\text{STAE}} = \|A(x) - S'(x)\|^2$
설계 의도 하나의 Student 네트워크가 두 가지 "정상의 모습"을 동시에 학습. 추론 시 두 출력 모두를 사용해 (구조적 이상) + (논리적 이상)을 한 번에 계산.

3. 전체 구조 — 3개 네트워크 명확화

입력 이미지 x
      │
      ├──→ Teacher PDN  (frozen, ImageNet 사전학습)         → T(x)   shape (384, h, w)
      │
      ├──→ Student PDN  (학습됨, random init)               → S(x)   첫 384채널: Teacher 흉내
      │                                                     → S'(x)  뒤 384채널: AE 흉내
      │
      └──→ Autoencoder  (학습됨, PDN과 다른 구조!)           → A(x)   shape (384, h, w)
                       (encoder + decoder, 작은 latent dim)


[추론 시 score 계산]

  M_local  = ||T(x) - S(x) ||²       — 구조적 이상 (작은 결함)
  M_global = ||A(x) - S'(x)||²       — 논리적 이상 (전역 구조)
  M        = 0.5 * (norm(M_local) + norm(M_global))
자주 혼동되는 부분 "Autoencoder도 PDN의 일종인가?" — 아닙니다. Autoencoder는 별도의 encoder-decoder 구조로, PDN보다 작은 receptive field와 latent dimension을 가집니다. Autoencoder는 "전역 정보를 압축해서 정상 평균에 가까이 모이도록" 설계된 다른 네트워크.

4. 학습 방식의 차이

네트워크가중치 초기화학습되는가?학습 데이터
Teacher PDN ImageNet에서 사전학습된 가중치 로드 (논문 저자 제공) ❌ frozen (없음 — 사전학습 시 ImageNet 사용)
Student PDN random ✓ 학습됨 대상 제품 정상 이미지만
Autoencoder random ✓ 학습됨 대상 제품 정상 이미지만

Teacher의 ImageNet 사전학습이 특이한 점

EfficientAD 논문은 Teacher PDN을 ImageNet에서 그냥 supervised 학습한 것이 아닙니다. WideResNet-101 같은 큰 ImageNet 모델로부터 distillation해서 만들었습니다. 즉:

WideResNet-101 (huge, ImageNet trained)
         │
         │  distillation (PDN이 같은 feature를 만들도록 학습)
         ▼
   Teacher PDN (small, ImageNet feature 압축본)

이렇게 만들어진 Teacher PDN을 anomalib 또는 논문 저자가 weight 파일로 배포 → 우리는 그걸 받아서 그대로 frozen으로 사용.

5. PyTorch 코드 — 명확하게 구분

import torch
import torch.nn as nn

# ===== PDN 아키텍처 (Teacher와 Student 공통) =====
def build_pdn(out_channels):
    """Teacher: out_channels=384, Student: out_channels=768"""
    return nn.Sequential(
        nn.Conv2d(3,   128, 4, stride=1), nn.ReLU(),
        nn.AvgPool2d(2, 2),
        nn.Conv2d(128, 256, 4, stride=1), nn.ReLU(),
        nn.AvgPool2d(2, 2),
        nn.Conv2d(256, 256, 3, stride=1), nn.ReLU(),
        nn.Conv2d(256, out_channels, 4, stride=1),   # ← 여기만 다름!
    )

# Teacher: 384 채널, ImageNet 사전학습 weight 로드, frozen
teacher = build_pdn(out_channels=384)
teacher.load_state_dict(torch.load("teacher_small.pth"))   # 논문 제공
teacher.eval()
for p in teacher.parameters():
    p.requires_grad = False

# Student: 768 채널 (= 384 ST 흉내 + 384 AE 흉내), random init, 학습됨
student = build_pdn(out_channels=768)
student.train()

# Autoencoder: 완전히 다른 구조 (PDN 아님)
autoencoder = nn.Sequential(
    # Encoder (입력을 작은 bottleneck으로 압축)
    nn.Conv2d(3,  32, 4, stride=2, padding=1), nn.ReLU(),
    nn.Conv2d(32, 64, 4, stride=2, padding=1), nn.ReLU(),
    nn.Conv2d(64, 64, 4, stride=2, padding=1), nn.ReLU(),
    nn.Conv2d(64, 64, 4, stride=2, padding=1), nn.ReLU(),
    nn.Conv2d(64, 64, 4, stride=2, padding=1), nn.ReLU(),
    nn.Conv2d(64, 64, 8, stride=1),                          # bottleneck
    # Decoder (384 채널 출력으로 복원 — Teacher feature와 매칭)
    nn.ConvTranspose2d(64, 64, 4, stride=2, padding=1), nn.ReLU(),
    # ... (여러 ConvTranspose 층)
    nn.Conv2d(64, 384, 3, stride=1, padding=1),              # 최종 384채널 출력
)
정리 PDN은 Teacher와 Student가 같은 아키텍처. 출력 채널만 다름 (384 vs 768). Autoencoder는 완전히 다른 구조.

6. 왜 Student와 Teacher가 같은 아키텍처여야 하는가?

이론적으로 Student와 Teacher의 아키텍처가 달라도 distillation은 가능합니다. 그러나 EfficientAD가 일부러 동일 아키텍처를 쓰는 이유:

  1. 완벽한 흉내 가능성 보장 — 같은 아키텍처면 Student가 정상 patch에 대해 Teacher 출력을 이론상 완벽히 흉내낼 수 있음. 차이는 학습 부족 때문, 아키텍처 한계 때문이 아님
  2. 이상 검출 신호의 의미가 명확 — Student가 못 따라가면 "Student가 정상만 봐서 이상을 흉내 못 냄" 이라고 해석 가능. 아키텍처 차이로 인한 표현력 한계가 섞이지 않음
  3. Latency 동일 — 추론 시 두 forward의 비용이 같음 → 총 latency 예측 가능
  4. Hard Feature Loss와의 정합성 — top-K patch에서의 차이가 순수하게 "학습 부족"으로 해석됨

만약 Student가 더 작은 네트워크라면 정상도 못 흉내내는 영역이 생겨 학습 신호가 흐려짐. 만약 더 크다면 over-distillation 위험 + latency 증가.

7. PatchCore의 backbone과의 비교

측면PatchCoreEfficientAD
Backbone 1개?O — WideResNet-50 (frozen)X — Teacher PDN + Student PDN + Autoencoder (3개)
Backbone 학습?X (ImageNet pretrained 그대로)Teacher는 frozen, 그러나 Student + AE는 학습
학습 비용거의 0 (memory bank 구축만)epoch 단위 학습 필요 (수 시간)
추론 forward 횟수1회 (backbone) + 1회 (kNN)3회 (Teacher, Student, AE)

PatchCore와 달리 EfficientAD는 여러 네트워크의 협업으로 동작합니다 — backbone 하나가 모든 일을 하는 게 아닙니다.

latency 측정 시 포함해야 할 항목

  • Autoencoder forward (한 번)

논문의 "2 ms/img"는 위 모두를 포함한 수치 (RTX A6000 기준). PatchCore의 "수치 불명" latency와 비교 시 동등 조건 보장 필수.

9. 결론

최종 답
  • Teacher PDN과 Student PDN은 같은 아키텍처 (Conv1~3 동일)
  • 마지막 Conv4 출력 채널만 다름: Teacher 384, Student 768 (Student는 두 흉내 task)
  • 학습 방식과 가중치 초기화는 다름: Teacher는 사전학습 후 frozen, Student는 random + 학습
  • Autoencoder는 PDN이 아닌 별도의 encoder-decoder 구조
