5개 모델은 PatchCore와 같은 방식인가? — 패러다임 분류

질문자: 연구자 (사용자) 작성일: 2026-05-27 관련: 분석 ① 백본 도메인 적응

0. 한 줄 답

정정 (이전 보고서의 묶음에 대한 명확화) 아닙니다. 5개 모델은 모두 다른 패러다임입니다. 이전 보고서(분석 ①)에서 한 표에 묶었던 이유는 "ImageNet 백본 적응이라는 문제에 대한 답"이라는 공통 주제 때문. 실제 동작 방식은 PatchCore와 직접 같은 건 AnomalyDINO 하나뿐이고, CFA는 변형, 나머지 3개는 다른 패러다임.

1. PatchCore의 패러다임을 다시 정의

"PatchCore와 같은 방식"이 무엇인지 먼저 명확히 합니다. PatchCore는 다음 4가지 특징을 모두 가진 패러다임:

  1. Pretrained encoder를 그대로 사용 (학습 안 함) — 학습 비용 0
  2. 정상 patch feature를 저장한 메모리 뱅크 — coreset subsampling으로 1% 유지
  3. k-NN 거리 기반 scoring — 테스트 patch와 메모리 뱅크의 최근접 거리
  4. 학습 가능한 파라미터 0 (decoder, classifier, hypersphere 등 없음)

$$ S(x) = \max_{f^* \in \text{patches}(x)} \min_{m \in M} \|f^* - m\|_2 $$

이 4가지를 모두 만족하면 "PatchCore와 같다"고 할 수 있고, 일부만 만족하면 "PatchCore-인접" 또는 "다른 패러다임"입니다.

2. Anomaly Detection 패러다임 4분류

학계에서 unsupervised AD는 다음 4개 큰 갈래로 나뉩니다.

패러다임 핵심 아이디어 대표 모델 이 보고서의 5개
Memory-bank / k-NN 정상 feature를 저장 → 테스트 거리 PatchCore, PaDiM, SPADE AnomalyDINO ✓, CFA (변형)
Knowledge Distillation Student가 Teacher 흉내내도록 학습 → 차이로 score EfficientAD, STFPM, RD4AD ReContrast (contrastive 추가)
Reconstruction-based 정상을 잘 재구성 가능 → 이상은 재구성 실패 AutoEncoder, RIAD Dinomaly (feature-level 재구성)
Synthesis-based / Self-supervised 가짜 결함 합성 → 분류기 또는 segmentation 학습 DRAEM, SimpleNet, GLASS CutPaste ✓
한눈에 보는 결론 - AnomalyDINO만이 PatchCore와 거의 동일 (백본만 DINOv2로 교체)
- CFA는 PatchCore와 인접 (메모리 뱅크 대신 학습 가능한 hypersphere)
- ReContrast는 KD 패러다임 (PatchCore와 본질 다름)
- Dinomaly는 Reconstruction 패러다임 (PatchCore와 본질 다름)
- CutPaste는 Synthesis 패러다임 (PatchCore와 본질 다름)

3. 모델별 상세 비교

3.1 AnomalyDINO — PatchCore의 직계 (백본만 교체)

속성AnomalyDINO
패러다임Memory-bank / k-NN
EncoderDINOv2 ViT (frozen, 학습 안 함)
학습 가능 파라미터0 (PatchCore와 동일)
메모리 뱅크O — 정상 patch feature 저장
Score 공식$ S(x) = \max \min \|f - m\|_2 $ — PatchCore와 완전 동일
PatchCore와의 차이딱 한 줄: backbone = ImageNet WideResNet → backbone = DINOv2 ViT
결론 "PatchCore의 백본을 DINOv2로 바꾼 것" — 사실상 PatchCore의 변종. 분석 ①의 (라) 옵션 그 자체.

3.2 CFA — PatchCore-인접 (메모리 뱅크를 학습 가능한 hypersphere로 대체)

속성CFA
패러다임Memory-bank의 학습 가능한 변형
EncoderPretrained (WideResNet 등, frozen)
학습 가능 파라미터O — Target-oriented Feature adaptation 모듈 (TFM) + 학습 가능한 hypersphere centers
메모리 뱅크대체됨 — 정적 뱅크 대신 학습된 hypersphere $C = \{c_1, ..., c_K\}$
학습 loss - "Attract": 정상 feature를 가까운 hypersphere로 끌어당김
- "Repel": hard negative로 hypersphere끼리 분리 (collapse 방지)
Score 공식$ S(x) = \max \min \|\phi(f) - c_k\|_2 $ — $\phi$는 TFM, $c_k$는 학습된 center
PatchCore와의 차이scoring 공식의 모양은 같지만, $M$이 학습으로 압축된 중심점 K개로 대체. 그리고 TFM이라는 adaptation layer가 추가됨
결론 PatchCore와 scoring 메커니즘은 매우 유사 (거리 기반), 하지만 학습 가능한 adaptation이 들어감. "PatchCore + 학습된 메모리 뱅크" 정도의 변형.

3.3 ReContrast — KD 패러다임 (PatchCore와 본질 다름)

속성ReContrast
패러다임Knowledge Distillation (RD4AD 계열) + Contrastive
구성Encoder (pretrained, 적응 가능) + Bottleneck + Decoder
학습 가능 파라미터O — Encoder 적응 + Decoder 전체 학습
메모리 뱅크없음
학습 loss - Reconstruction: Decoder가 Encoder feature 재구성
- Contrastive: target-domain feature가 source(ImageNet)와 다르게 학습되도록
Score 공식$ S(x) = \sum_l \|E_l(x) - D_l(x)\|^2 $ — encoder vs decoder의 feature 거리
PatchCore와의 차이패러다임 자체가 다름. 정상 feature를 저장하는 게 아니라, encoder-decoder 쌍을 학습해서 그 둘의 mismatch로 anomaly 판정. EfficientAD의 Teacher-Student 관계와 더 비슷.
결론 ReContrast는 EfficientAD의 KD 패러다임에 가깝다. PatchCore와는 메모리 뱅크 / k-NN 부분이 본질적으로 다름.

3.4 Dinomaly — Reconstruction 패러다임 (PatchCore와 본질 다름)

속성Dinomaly
패러다임Feature-level Reconstruction
구성DINOv2 ViT Encoder (frozen) + Linear-attention Bottleneck + ViT Decoder
학습 가능 파라미터O — Bottleneck + Decoder (Encoder는 frozen)
메모리 뱅크없음
학습 loss"Loose reconstruction" — Decoder가 Encoder feature를 재구성하되 완벽하게는 안 함 (over-fitting 방지)
Score 공식$ S(x) = \|E(x) - D(B(E(x)))\|^2 $ — encoder feature와 decoder 재구성의 거리
PatchCore와의 차이완전 다른 패러다임. "정상은 잘 재구성된다, 이상은 재구성 실패" 가정. 메모리 뱅크 없음.
결론 PatchCore와 공유하는 건 "DINOv2 같은 강한 backbone을 쓴다"는 점뿐. 메커니즘은 본질 다름. ReContrast와 가까운 구조이지만 contrastive 대신 loose reconstruction 사용.

3.5 CutPaste — Synthesis 패러다임 (PatchCore와 본질 다름)

속성CutPaste
패러다임Self-supervised classification with synthetic anomalies
구성Encoder + Classification head (2 class: normal vs CutPaste)
학습 가능 파라미터O — Encoder 전체 학습 (혹은 fine-tune)
메모리 뱅크없음 (학습 시) — inference 시 GDE 또는 kNN으로 보조 가능
학습 절차 1. 정상 이미지 일부를 잘라 다른 위치에 붙임 → "합성 결함"
2. Encoder가 (정상 vs 합성결함) 분류하도록 학습
3. 분류 task로 학습된 feature가 anomaly 검출에도 유용해짐
Score 공식다양 — classifier 출력의 softmax, 또는 학습된 feature에 GDE/kNN 적용
PatchCore와의 차이학습 패러다임이 완전 다름. "합성 결함을 만들고 그것을 구분하는 task로 좋은 feature를 얻는다" — PatchCore의 "정상 feature 저장 + 거리"와는 본질적으로 다른 접근.
결론 패러다임이 완전 다르다. 다만 CutPaste의 결과물(학습된 encoder)을 PatchCore의 백본으로 쓰는 것은 가능 — 그것이 분석 ①의 (다) 옵션.

5. 이전 보고서에서 왜 묶었는가

분석 ①에서 5개 모델을 같은 표에 둔 이유는 모두 "ImageNet 백본 적응" 문제에 답을 제시하기 때문이었습니다. 각 모델이 그 문제에 어떻게 답하는지 정리하면:

모델"백본 적응"에 대한 답PatchCore에 적용 가능?
AnomalyDINO "백본을 DINOv2로 교체" ✓ 즉시 (1줄 코드)
CFA "백본 위에 학습 가능한 hypersphere adaptation 추가" ✓ TFM 모듈만 추가 (학습 필요)
CutPaste "합성 결함 task로 백본을 self-supervised 학습 → 그 백본을 다른 모델에 활용" ✓ 학습 결과 백본을 PatchCore에 꽂으면 됨
ReContrast "contrastive로 백본을 target domain에 적응" △ ReContrast 자체는 다른 패러다임, 학습된 백본만 차용 가능
Dinomaly "DINOv2를 활용 + reconstruction 패러다임 자체로 우월" △ Dinomaly 자체는 다른 모델, DINOv2 backbone 아이디어만 차용 가능
정확한 표현 이전 보고서의 "보고 개선치"는 각 모델 자체의 성능이지 "PatchCore + 그 모델의 백본 아이디어"의 개선치가 아닙니다. 본 연구에서 정확하게 측정하려면 다음과 같이 분리해야 합니다.
  • PatchCore vs PatchCore-DINOv2 백본 (AnomalyDINO 변형) — 같은 패러다임에서 backbone 효과 측정
  • PatchCore vs PatchCore + CutPaste-pretrained backbone — synthesis-pretrained backbone 효과 측정
  • PatchCore vs CFA — 메모리 뱅크 학습화 효과 측정
  • PatchCore vs Dinomaly — 메모리 뱅크 vs reconstruction 패러다임 비교
  • PatchCore vs ReContrast — 메모리 뱅크 vs KD 패러다임 비교

6. 본 연구 P1에의 의미

P1 IF-PROTO의 5종 SOTA 비교가 이 패러다임 다양성을 의도적으로 포함해야 합니다.

패러다임P1 대표 모델 (제안)이유
Memory-bank (정적) PatchCore (ImageNet) — baseline 산업 표준 기준점
Memory-bank (정적, DINOv2) AnomalyDINO 또는 PatchCore-DINOv2 변형 backbone 단일 변수 효과 측정
Knowledge Distillation EfficientAD latency 극단 + KD 대표
Reconstruction Dinomaly 그룹 A의 SOTA
Synthesis GLASS 또는 DRAEM 합성 결함 패러다임 대표
제안 P1을 "5종 SOTA"가 아니라 "패러다임 4종 + backbone 1변수"로 재구성하면, recall@FPR=0.1% 측정이 패러다임 자체에 대한 비교 + 변수 분리 효과 모두 제공. 본 연구의 핵심 기여 강화.

7. 결론 (다시 한 줄)

정정된 답 - AnomalyDINO: PatchCore와 거의 같음 (backbone만 다름)
- CFA: PatchCore와 인접 (학습 가능한 adaptation 추가)
- ReContrast / Dinomaly / CutPaste: PatchCore와 다른 패러다임
이전 보고서의 묶음은 "백본 적응 문제에 대한 각자의 답"이라는 공통 주제 때문이었지, 실제 모델 동작은 같지 않음. 본 연구의 5종 비교는 패러다임 다양성을 의도적으로 반영하도록 재구성하는 것이 학술·산업 모두 가치 높음.
