PatchCore의 ImageNet 백본을 대상 제품 데이터로 추가 학습 — 도움이 될까?

질문자: 연구자 (사용자) 작성일: 2026-05-27 관련: 학습 ① PatchCore

0. 한 줄 답

결론 (선요약) 방식에 따라 도움이 되기도, 오히려 해가 되기도 한다. "정상 이미지로 backbone을 그냥 fine-tune" 하는 가장 직관적인 방식은 PatchCore에서 정확도를 떨어뜨릴 위험이 큼 (feature collapse). 반면 self-supervised 방식이나 합성 결함을 활용한 contrastive 방식은 보고된 개선치가 있음 (0.5~3%p 수준).

이 보고서는 다음 순서로 진행:

  1. 질문의 두 갈래 정리
  2. 왜 "정상만 fine-tune"이 위험한가 — feature collapse 원리
  3. 보고된 사용 케이스 (논문 + 산업)
  4. 의견 (조건부 추천 + 우선순위)
  5. 본 연구에 어떻게 검증할까

1. 질문의 두 갈래

"대상 제품 데이터로 추가 학습"이라는 한 문장에 사실 두 가지 매우 다른 방법이 섞여 있습니다.

방식구체적으로위험도
(가) 정상 데이터만으로 supervised fine-tune ImageNet pretrained 가중치를 시작점으로, 대상 제품의 정상 이미지에 classification/reconstruction loss 같은 일반 학습 고위험 — feature collapse 가능성
(나) 자기지도 학습 (Self-Supervised) on target DINO, MoCo, SimCLR, MAE 등으로 대상 제품 이미지 (정상/결함 무관)로 backbone 자기지도 학습 중위험 — 데이터·시간 비용, 효과는 도메인 차이에 비례
(다) 합성 결함 기반 contrastive CutPaste / DRAEM 식으로 정상에서 가짜 결함을 만들고, 정상↔합성결함을 구분하도록 학습 중위험 — 합성 분포가 실제와 어긋날 수 있음
(라) Foundation model 교체 backbone을 ImageNet supervised에서 DINOv2 self-supervised로 교체 (학습 안 함, 가중치만 교체) 저위험 — 가장 안전·실효성 검증된 경로

대부분의 "fine-tune이 도움이 되는가" 질문은 사실 (가)를 의도합니다. 그러나 보고된 개선치는 대부분 (나)~(라)에서 나옵니다.

2. 왜 "정상만 fine-tune"은 위험한가 — Feature Collapse

PatchCore의 score 공식을 다시 봅시다.

$$ S(x) = \max_{f^* \in \text{patches}(x)} \min_{m \in M} \|f^* - m\|_2 $$

여기서 $M$은 정상 patch feature의 memory bank. 이 score가 "이상"을 잘 잡으려면 다음 두 가지가 동시에 성립해야 합니다.

  1. 정상끼리는 가깝다 — 같은 정상 patch는 feature space에서 비슷한 위치에 모임
  2. 이상은 정상에서 멀다 — 이상 patch는 정상 cluster 밖으로 떨어짐

ImageNet pretrained는 1000개 클래스를 구분하도록 학습됐기 때문에 feature space에 "다양성"이 보존돼 있음. 정상 제품 이미지가 그 다양한 공간의 일부에 자리잡고, 결함은 그 자리에서 벗어나는 방향으로 떨어집니다.

정상만 fine-tune의 함정 정상 이미지만으로 backbone을 fine-tune하면, 모델은 "이 모든 입력을 비슷한 feature로 매핑"하는 방향으로 학습됩니다. 왜냐하면 정상 이미지들끼리는 실제로 비슷하기 때문이죠. 결과:
  • 정상 cluster가 매우 좁아짐 (compression)
  • 이상 이미지도 학습에서 본 적 없으니 똑같이 그 좁은 cluster로 매핑될 가능성이 있음 (모델이 "다른 것을 다르게" 표현할 이유가 없으니까)
  • 결과적으로 ${\|f_{\text{이상}} - M\|}$이 작아짐 → 이상 score가 작아짐 → AUROC ↓

이것을 학계에서는 feature collapse (또는 representation collapse)라고 부릅니다. self-supervised learning(BYOL, SimSiam) 논문들이 이 문제를 우회하기 위해 stop-gradient, predictor, EMA 같은 트릭을 도입한 이유입니다.

PatchCore 원 논문(Roth et al., CVPR 2022, §4.4)도 ablation에서 "ImageNet pretrained 그대로가 가장 좋다"고 보고합니다. 다른 fine-tuned backbone들은 모두 성능 하락.

3. 보고된 사용 케이스

3.1 학계 — 효과를 입증한 방법

접근 논문 방식 분류 보고 개선 (MVTec) 핵심 트릭
CFA Lee et al., Coupled-hypersphere-based Feature Adaptation for Target-Oriented Anomaly Localization, IEEE Access 2022, arXiv:2206.04325 (나)+(다) 혼합 I-AUROC 99.5% (PatchCore 99.2 → +0.3%p) 정상 patch를 hypersphere로 압축하되, hard negative mining으로 다양성 보존
CutPaste Li et al., CVPR 2021, arXiv:2104.04015 (다) I-AUROC 96.1% (Original 자기지도 학습으로 backbone 학습) 이미지 일부를 잘라 다른 위치에 붙여 "합성 결함" 생성 → 정상 vs 합성을 구분하는 self-supervised
ReContrast Guo et al., NeurIPS 2023, arXiv:2306.02602 (나) I-AUROC 99.5% (RD4AD 98.5 → +1.0%p) contrastive learning으로 target-domain encoder/decoder 동시 학습. backbone 전체 적응
DRAEM Zavrtanik et al., ICCV 2021, arXiv:2108.07610 (다) I-AUROC 98.0% Perlin noise + DTD 텍스처로 합성 결함 → encoder 학습. PatchCore와 패러다임 다름이지만 핵심 아이디어 공통
Dinomaly Guo et al., CVPR 2025, arXiv:2405.14325 (라) — DINOv2 사용 I-AUROC 99.6~99.8% backbone 교체가 가장 큰 기여. ImageNet supervised → DINOv2 self-supervised → +0.4~0.6%p
AnomalyDINO Damm et al., WACV 2025, arXiv:2405.14529 (라) PatchCore baseline 대비 zero-shot에서 +5~10%p (cross-domain) DINOv2 그대로, k-NN 기반 score. ImageNet 대비 cross-domain transfer 우수

3.2 학계 — "정상만 fine-tune"이 실패한 보고

  • PatchCore 원 논문 (Roth et al., 2022, §4.4) — backbone fine-tune ablation에서 모두 성능 하락 보고
  • Domain-adapted ResNet for AD (다수 워크숍 보고) — supervised fine-tuning이 baseline 대비 0.5~3%p 하락 사례 다수
  • PaDiM ablation — 정상 데이터에 추가 supervised 학습 시 다변량 정규분포의 분산이 0에 가까워져 score 무의미해짐

3.3 산업 보고 (블로그·케이스 스터디 수준)

출처보고 내용신뢰도
Intel × Anomalib 블로그 "ImageNet feature가 대부분의 제조 도메인에 충분. 도메인 변경(X-ray, IR) 시에만 추가 학습 필요" 중간 (선언적)
MVTec 공식 매뉴얼 (HALCON/MERLIC) backbone fine-tune 미권장. 대신 데이터 증강·해상도 조정 권장 높음 (상용 제품 결과)
BMW·Bosch 케이스 스터디 (CVPR/WACV 발표) 구체적 수치 비공개. "feature extractor는 일반적인 것 사용, 도메인 특화는 후처리에서" 흐름 중간 (요약 발표)
X-ray 검사 (보안·의료) 연구 도메인 격차가 클 때 self-supervised pretraining 필수, 보고된 개선 5~15%p 높음 (다수 논문)
패턴 요약 "가시광 RGB 산업 검사" (MVTec 같은 일반 제조)에서는 ImageNet feature가 이미 강력해서 추가 학습 효과가 작음. RGB가 아닌 modality나 매우 특수한 외관일수록 추가 학습 효과 큼.

4. 의견 — 조건부 추천

이하는 위 문헌과 본 연구의 제약 조건을 종합한 권고입니다. 확률은 정성적 신뢰도입니다 — 본 연구에서 검증할 수치는 자체 측정으로 확정해야 합니다.

4.1 의사결정 트리

대상 제품이 가시광 RGB인가?
├── Yes (대부분의 제조 검사)
│   │
│   대상 제품이 ImageNet 클래스와 외관 유사한가? (음식, 부품, 가전 등)
│   ├── Yes → backbone 추가 학습 비추천. ImageNet 그대로 + 데이터 증강·해상도·calibration에 시간 투자
│   │         예상 개선치: 0~0.5%p (CI 안에 묻힐 가능성 높음)
│   │
│   └── No (특수 텍스처, 미세 패턴, 비전형 외관)
│       → (다) 합성 결함 기반 contrastive 또는 (라) DINOv2 교체 추천
│         예상 개선치: 0.5~2%p
│
└── No (X-ray, IR, 적외선, 초음파, 마이크로스코프, 하이퍼스펙트럴)
    → 추가 학습 강력 추천. (나) self-supervised on target 우선
      예상 개선치: 5~15%p (도메인 격차가 크므로)

4.2 우선순위 (시간 대비 효과)

순위접근예상 개선구현 난이도총평
1 backbone을 DINOv2로 교체 (라) 0.4~1%p (RGB), 더 큰 도메인 격차에선 ↑ 매우 낮음 (1줄 코드 변경) 가장 ROI 높음. 본 연구에서 PatchCore baseline 외에 PatchCore-DINOv2도 측정 강추
2 합성 결함 + contrastive (다, CutPaste/CFA 식) 0.3~1.5%p 중간 이미 anomalib에 CutPaste, DRAEM 구현 있음. 학습 1~3시간 추가
3 self-supervised on target (나, DINO/MAE) 0~1%p (RGB), 5~15%p (특수 도메인) 높음 대상 도메인이 ImageNet과 매우 다를 때만. 학습 12~48시간
4 정상만 supervised fine-tune (가) -1~-3%p (악화 가능) 중간 (위험) 비추천. feature collapse 위험. 권장하지 않음.

4.3 본 연구 관점에서의 권고

권고
  1. 먼저 (라) DINOv2 교체를 P1 evaluation protocol의 비교군에 추가 — 1줄 코드 변경으로 ImageNet vs DINOv2 backbone PatchCore 비교 가능
  2. 만약 대상 제품이 "특수 도메인" (X-ray 등)이라면 (나) self-supervised pretraining이 본 연구의 핵심 기여로 격상 가능 — Top-2 한계(도메인 시프트)의 직접 답
  3. (가) "정상만 fine-tune"은 본 연구에서 검증 실패 케이스로 보고 — feature collapse 경험적 입증이 부정적 결과 자체로 학술 기여

5. 본 연구에서의 검증 방법

본 연구의 P1 IF-PROTO는 이 질문을 검증할 적합한 도구를 이미 갖추고 있습니다.

검증 항목실험 설계측정값
(가) 위험성 입증 ImageNet PatchCore vs Normal-finetune PatchCore (10 epoch) recall@FPR=0.1% + bootstrap CI · 가설: finetune 쪽이 같거나 낮음
(라) DINOv2 효과 ImageNet PatchCore vs DINOv2 PatchCore — 동일 카테고리 recall@FPR + CI · per-defect-size recall (특히 미세결함)
(다) 합성 결함 기여 ImageNet PatchCore vs CutPaste-pretrained PatchCore recall · CI
도메인 격차 효과 MVTec(가시광 일반) vs MVTec AD 2 (특수 결함) — 두 데이터셋에서 (가)~(라) 효과 비교 각 데이터셋에서의 상대 개선치
Bootstrap CI 주의 Phase 4에서 발견한 측정 노이즈에 의해, MVTec 단일 카테고리 규모에서는 0.5~1%p 개선이 CI 안에 묻힐 가능성이 큽니다. backbone 차이 측정은 15-카테고리 pooled 또는 Real-IAD scale에서 하는 것이 신뢰성 ↑.

6. 결론 (한 줄)

최종 답 "정상 데이터만으로 ImageNet backbone을 그냥 추가 학습"은 거의 항상 비추천. 동일한 의도(대상 도메인 적응)를 달성하려면 DINOv2 교체 → 합성 결함 contrastive → self-supervised pretraining 순으로 시도. 본 연구는 이 비교를 P1 IF-PROTO에 추가하기에 자연스러운 구조.
