Soft label에서 CE 대신 KL을 쓰는 이유 — "둘 다 정보량 측정 아닌가?"

질문자: 연구자 (사용자) 작성일: 2026-05-28 관련: 개념 ② · 개념 ③ · 분석 ⑤

0. 한 줄 답

결론 학습 dynamics는 동일. KL을 쓰는 이유는 4가지 — 수학이 아닌 실용/표현/도구의 영역:
  1. 의미가 명확 — "분포 매칭"인지 "분류"인지가 코드로 드러남
  2. 0이 기준점 — 학습이 얼마나 진행됐는지 절대적으로 알 수 있음
  3. 다중 loss 결합 시 깔끔 — Hinton KD의 $\alpha$ 가중치 해석이 명료해짐
  4. API가 사실상 강제 — PyTorch F.cross_entropy는 soft target 못 받음
사용자의 직관 "둘 다 정보량 측정 아닌가?"는 맞지만 정밀히 보면 측정 대상이 다름: CE는 "절대 비용", KL은 "비효율의 거리". 차이를 아는 게 두 함수의 선택 기준.

1. 사용자의 질문 분해

지금까지 정리:

사실확인 위치
One-hot에서 $H(p) = 0$ (convention 하에서) → CE = KL exact분석 ⑤
Hard label은 CE를 쓴다 (정답 인덱스 하나만, 효율)개념 ②
Soft label에서도 gradient는 동일개념 ②, ③

사용자의 자연스러운 후속 질문: "그럼 soft label에서도 CE를 쓰면 안 되나? 왜 굳이 KL?"

이 질문은 정확히 우리가 답해야 할 것입니다.

2. 정말 같은 것을 측정하는가? — 측정 대상의 차이

사용자의 표현 "둘 다 정보량 평가"는 큰 틀에서는 맞지만, 정밀하게 보면 측정 대상이 다릅니다.

양측정 대상의미
$H(p)$ $p$ 자체의 무질서도 "내가 가진 데이터에 본질적으로 있는 불확실성"
$H(p, q)$ $p$ 데이터를 $q$ 코드로 인코딩한 평균 비용 "잘못된 가정의 코드로 데이터를 표현한 총 비용 (절대값)"
$D_{KL}(p \| q)$ $p$와 $q$ 사이의 비효율 거리 "내가 잘못 가정해서 추가로 지불한 비용 (상대값, 0이 최적)"

$H(p, q)$와 $D_{KL}$의 관계는:

$$ H(p, q) = \underbrace{H(p)}_{\text{본질적 부분}} + \underbrace{D_{KL}(p \| q)}_{\text{비효율 부분}} $$

결정적 차이
  • CE는 혼합된 양: 데이터 자체의 본질 ($H(p)$) + 모델의 비효율 ($D_{KL}$)을 같이 잰다.
  • KL은 순수한 양: 모델의 비효율만 잰다. 데이터 자체의 본질은 빠져있다.
"같은 정보를 잰다"라기보다 "같은 큰 그림 안의 서로 다른 부분"을 잰다가 정확.

비유 — 음식 배달 비용

음식을 주문해서 배달받는 상황을 상상해봅시다.

요소의미
음식 자체 가격 = $H(p)$변할 수 없는 본질적 비용
배달비 = $D_{KL}(p \| q)$잘못된 배달 회사 (q)를 선택해서 추가로 든 비용. 직거래라면 0
총 지불 금액 = $H(p, q)$음식값 + 배달비 — 총 비용

학습 = 더 좋은 배달 회사 찾기. 즉 배달비($D_{KL}$)를 0에 가깝게 하는 것. 학습으로 영향 줄 수 있는 부분은 배달비 ($D_{KL}$)뿐이고, 음식값 ($H(p)$)은 외생적으로 고정.

총 비용 (CE)으로 학습 진행을 보려면 음식값을 빼야 함 → 결국 KL을 보는 것과 같음. KL은 음식값을 빼고 배달비만 직접 보여주는 metric.

3. 왜 KL이 더 자주 선택되는가 — 4가지 이유

3.1 의미적 명확성 — 코드의 의도

코드를 읽는 사람에게:

# 신호: "분포 매칭을 한다"
loss = F.kl_div(log_q_student, p_teacher, reduction='batchmean')

# 신호: "분류기 학습을 한다"
loss = F.cross_entropy(student_logits, y_hard_label)

같은 일을 하더라도 의도가 다르면 함수를 분리. 미래의 코드 읽는 사람 (또는 6개월 후의 나)이 즉시 "이건 KD soft target 매칭이다"라고 알 수 있음.

3.2 0이 기준점 — 학습 진행 모니터링

가장 결정적인 차이. 학습 도중 loss 곡선을 볼 때:

함수최솟값"수렴했는가?" 판단
$H(p, q)$ (CE) $H(p)$ (양수, $p$에 따라 다름) $H(p)$를 모르면 loss=0.5가 좋은지 나쁜지 알 수 없음
$D_{KL}(p \| q)$ (KL) 0 (모든 $p$에 대해) loss=0.5는 "정답에서 0.5만큼 떨어졌다"는 절대적 정보

실용적 가치 실험을 100개 돌릴 때, KL을 쓰면 "0.05 미만이면 잘 학습됨"이라는 절대적 기준을 만들 수 있음. CE를 쓰면 각 실험마다 $H(p)$가 달라 절대적 기준 불가.

3.3 다중 loss 결합 시 가중치 해석

Hinton KD의 표준 loss:

$$ \mathcal{L} = \underbrace{(1-\alpha) \cdot \mathcal{L}_{\text{hard}}}_{\text{정답 학습}} + \underbrace{\alpha \cdot T^2 \cdot \mathcal{L}_{\text{soft}}}_{\text{teacher 흉내}} $$

여기서 $\mathcal{L}_{\text{soft}}$를 KL로 쓰는 게 자연스러운 이유:

옵션$\alpha$의 의미
둘 다 CE 사용 두 절댓값을 가중 평균. $\mathcal{L}_{\text{hard}}$는 $H(p_{\text{hard}}) = 0$이고 $\mathcal{L}_{\text{soft}}$는 $H(p_{\text{soft}}) > 0$. $\alpha = 0.5$여도 "절반씩 기여"가 아님 — soft 항이 항상 큼
Hard CE + Soft KL (표준) 두 항 모두 "오차에 비례" — 0이 정답. $\alpha$가 직관적으로 "soft 신호를 얼마나 신뢰하는가"를 뜻함

구체적 수치 비교

$p_{\text{hard}}$ one-hot, $p_{\text{soft}}$ teacher (예: $H = 1.14$), 학생이 완벽 학습 ($q = $ teacher) 상황:

항CE+CE 시CE+KL 시 (표준)
$\mathcal{L}_{\text{hard}}$ (CE)0 ($p$가 one-hot)0
$\mathcal{L}_{\text{soft}}$$H(p_{\text{soft}}) = 1.14$ (CE)0 (KL)
$\alpha = 0.5$ 시 총합$0.5 \cdot 0 + 0.5 \cdot 1.14 = 0.57$ — 학습 완료인데도 0 아님!$0.5 \cdot 0 + 0.5 \cdot 0 = 0$ — 학습 완료 신호 명확
중요한 관찰 CE+CE 결합은 "완벽 학습 시 loss = $\alpha \cdot H(p_{\text{soft}})$" 라는 의미 모호한 floor를 만든다. KL이 있으면 "완벽 학습 = 두 항 모두 0"으로 깔끔. 그래서 Hinton 원논문이 KL을 채택.

3.4 PyTorch API 강제 — 형식적 이유

PyTorch (그리고 TF, JAX 모두):

함수target 형식Soft target 가능?
F.cross_entropy(logits, target) 정수 인덱스 텐서 (B,) ❌ (PyTorch 1.10+ label_smoothing 인자 지원, 그러나 임의 분포는 불가)
F.cross_entropy(logits, prob_target) 분포 (B, K) (1.10+) ✓ (최근 추가, 그러나 KL과 동등)
F.kl_div(log_q, p) 분포 (B, K) ✓ (원래 설계)

역사적으로 soft target을 다루려면 KL을 쓸 수밖에 없었음. 라이브러리 API가 관례를 굳혔고, 그게 지금까지 이어짐.

4. 그럼 CE는 언제 쓰는가? — 명확한 가이드

상황추천이유
Target이 hard label (정수 인덱스) CE (F.cross_entropy) 효율적 — 정답 클래스만 $\log$ 계산. NaN 위험 없음
Hinton KD의 hard label 항 CE 위와 동일
Hinton KD의 soft label 항 KL 0이 기준점, $\alpha$ 가중치 해석 명확
Self-distillation, mutual learning KL 두 모델의 분포 거리를 학습 모니터링에 직접 사용
Label smoothing (soft한 정답) 둘 다 가능 $H(p_{\text{smooth}})$가 작아서 CE도 큰 문제 없음. 관행은 CE (smoothing 옵션 사용)
VAE/VI의 prior matching KL (의무) ELBO 공식에 KL이 명시적으로 등장
분포 거리 분석 / EDA KL "두 분포가 얼마나 다른가"를 직접 정량화

5. 학습 dynamics가 같다 — 다시 확인 + 시각적 직관

여기까지 KL을 옹호하는 이유가 많지만, "실제로 모델이 학습되는 방식"은 정확히 같다는 점을 잊으면 안 됩니다.

5.1 Gradient 동일

Student logit $z_j$에 대해:

$$ \frac{\partial H(p, q_s)}{\partial z_j} = q_{s,j} - p_j = \frac{\partial D_{KL}(p \| q_s)}{\partial z_j} $$

완전히 동일. 그래서 같은 $p, q$로 학습하면 weight update가 같음 → 학습 결과 같음.

5.2 그럼에도 다른 점 — 절댓값과 그 의미

            학습 진행 →

   값 (절댓값)
    ▲
1.5 │●━━━━━━━━━━━━━━━━━━━━━━━━━━━━●━━●━━●━━●  ← H(p) (변하지 않음)
    │
    │  ─────────────●─────●─────●─────●─────●  ← H(p,q) = CE
    │  ↘
1.0 │      ↘ "내려간다, 어디까지?" — H(p)에 수렴
    │           ↘
    │              ↘
0.5 │                ↘
    │                  ↘
    │                    ●─────●─────●─────●  ← D_KL(p||q) = KL
    │                      ↘
0   │                        ──────●  ← 0에 수렴 (완전 학습)
    └─────────────────────────────────────→ epoch

   두 곡선은 항상 H(p)만큼 거리.
   gradient는 같으므로 "내려가는 속도"는 같음.
   다른 건 "어디서 시작해서 어디로 가는가"뿐.
정리 같은 모델, 같은 데이터, 같은 학습 결과. 단지 loss 값을 "$H(p)$를 더한 채로 볼 것인가 (CE), 뺀 채로 볼 것인가 (KL)"의 차이.

6. 실전 코드 — 두 방식 비교

import torch
import torch.nn.functional as F

# Soft target (teacher output, 예시)
p = torch.tensor([0.7, 0.2, 0.07, 0.03])

# Student logits → softmax
z = torch.tensor([1.0, 0.5, 0.0, -0.3], requires_grad=True)
q = F.softmax(z, dim=-1)
log_q = F.log_softmax(z, dim=-1)

# === 방법 1: KL (표준) ===
loss_kl = F.kl_div(log_q, p, reduction='sum')
# print: tensor(0.0876, grad_fn=...)

# === 방법 2: Soft CE (직접 계산) ===
loss_ce = -(p * log_q).sum()
# print: tensor(1.0173, grad_fn=...)

# === 검증: H(p) ===
H_p = -(p * torch.log(p)).sum()
# print: tensor(0.9297)

# loss_kl == loss_ce - H_p?
print(loss_ce.item() - H_p.item())  # 0.0876 — KL과 일치 ✓

# gradient 비교
loss_kl.backward(retain_graph=True)
grad_kl = z.grad.clone()
z.grad.zero_()
loss_ce.backward()
grad_ce = z.grad.clone()

print(torch.allclose(grad_kl, grad_ce))  # True ✓
관찰 loss 값은 $H(p) = 0.93$만큼 다르지만 (KL = 0.09, CE = 1.02) gradient는 정확히 같음. 학습 결과 동일.

7. 본 연구로 다시

본 연구의 P1·P2는 classification distribution이 아닌 feature MSE를 사용하므로 CE/KL 선택 자체는 직접 적용되지 않습니다. 그러나 다음 의사결정에서 본 분석이 도움:

본 연구 영역본 분석의 시사
P2 CS-BRIDGE의 pseudo-label confidence weighting MuSc score의 entropy 일정 → KL로 비교 가능. P2 학습 신호의 진행 모니터링에 KL 직접 사용 권장
Calibration ECE 측정 (P1) ECE는 본질적으로 calibration의 KL과 관련. 0이 기준점이라는 KL의 장점이 그대로 적용
모델 ensemble의 발산 측정 두 모델의 score 분포 거리를 KL로 → ensemble 유효성을 직접 정량

8. 결론 (한 줄 — 다시)

최종 "둘 다 분포 사이의 정보 관련 양"이라는 직관은 옳습니다. 다만:
  • CE는 총 비용 (절대값) — $H(p)$가 포함된 혼합
  • KL은 비효율의 거리 (상대값) — 학습이 줄일 수 있는 부분만 순수하게 측정
같은 학습을 모니터링·표현·결합·인터페이스 4가지 측면에서 KL이 깔끔하므로 soft label에는 관례적으로 KL을 씁니다. 그러나 학습 결과는 어느 쪽을 쓰든 같다는 사실은 변하지 않습니다.

9. 4가지 이유 한눈에

이유한 줄중요도
1. 의미 명확성"분포 매칭" 신호를 코드로 표현중
2. 0 기준학습 진행을 절대적으로 판독 가능높음
3. Multi-loss 가중치$\alpha$ 해석이 명료해짐높음 (KD 핵심)
4. PyTorch APISoft target 입력에 자연스러운 인터페이스중
