왜 Hard label은 CE, Soft label은 KL을 쓰는가?

작성일: 2026-05-27 관련: 개념 ① Knowledge Distillation 학습 시간 20~30분

0. 한 줄 답 — 사실은 거의 같다

결론 수학적으로 두 손실은 같은 gradient를 만든다. 진짜 차이는:
  1. 입력 형식이 다름 — CE는 정답 클래스 인덱스 하나, KL은 전체 확률 분포
  2. 계산 효율 — CE는 정답 클래스 1개에만 $\log$ 계산 (수십~수천 배 빠름)
  3. 의도 표현 (semantic clarity) — "분류 학습"인지 "분포 매칭"인지 코드로 드러남
즉 "CE vs KL"의 구분은 본질이 아니라 구현 관행에 가깝습니다.

1. 정의 다시 보기

1.1 Cross-Entropy

분포 $p$ (정답)와 분포 $q$ (예측) 사이의 cross-entropy:

$$ H(p, q) = -\sum_{i=1}^{K} p_i \log q_i $$

$K$는 클래스 수.

1.2 KL Divergence

두 분포 사이의 "거리" (정확히는 거리가 아닌 비대칭 측도):

$$ D_{\text{KL}}(p \| q) = \sum_{i=1}^{K} p_i \log \frac{p_i}{q_i} $$

1.3 Entropy

분포 $p$ 자체의 무질서도:

$$ H(p) = -\sum_{i=1}^{K} p_i \log p_i $$

2. 핵심 항등식 — 두 손실의 관계

$D_{\text{KL}}$을 풀어 쓰면:

$$ D_{\text{KL}}(p \| q) = \sum_i p_i \log \frac{p_i}{q_i} = \sum_i p_i \log p_i - \sum_i p_i \log q_i $$

$$ = -H(p) + H(p, q) $$

정리하면:

기본 항등식 $$ \boxed{\; H(p, q) = D_{\text{KL}}(p \| q) + H(p) \;} $$ 즉, Cross-Entropy = KL Divergence + (target 분포의 Entropy)

이 한 줄이 모든 비밀을 담고 있습니다. Target $p$는 학습 동안 고정이므로 $H(p)$는 student 파라미터에 대해 상수 — gradient에 영향을 주지 않습니다.

$$ \nabla_{\theta} H(p, q_\theta) = \nabla_\theta D_{\text{KL}}(p \| q_\theta) \quad \text{(같은 gradient)} $$

3. Hard Label 경우 — 두 손실이 정확히 같다

Hard label은 one-hot 벡터입니다. 정답 클래스 $y$에 대해:

$$ p = (0, 0, \ldots, \underbrace{1}_{i=y}, \ldots, 0) $$

3.1 One-hot의 Entropy

$0 \log 0$은 극한으로 $0$이고, $1 \log 1 = 0$입니다. 따라서:

$$ H(p_{\text{one-hot}}) = -\sum_i p_i \log p_i = 0 $$

3.2 그러므로 CE = KL (완전히)

$$ H(p_{\text{one-hot}}, q) = D_{\text{KL}}(p_{\text{one-hot}} \| q) + 0 = D_{\text{KL}}(p_{\text{one-hot}} \| q) $$

의미 Hard label에서는 cross-entropy와 KL divergence가 수학적으로 같은 함수입니다. 우리가 CE를 쓰든 KL을 쓰든 결과는 동일.

3.3 그럼에도 왜 CE를 쓰는가?

One-hot 벡터에서 CE를 풀어 쓰면:

$$ H(p_{\text{one-hot}}, q) = -\sum_i p_i \log q_i = -1 \cdot \log q_y = -\log q_y $$

정답 클래스의 예측 확률에 $-\log$ 한 번만. KL은 형식적으로 $K$개 항을 합하지만 대부분 0이라 결과는 같음. CE는 그 효율을 명시적으로 활용합니다.

# CE: 인덱스 하나만 받아서 한 번의 로그 — 매우 효율적
loss_ce = -torch.log(q[y])

# KL: 형식상 K개 합을 계산 (대부분 0이지만)
loss_kl = torch.sum(p_onehot * (torch.log(p_onehot + eps) - torch.log(q)))

4. Soft Label 경우 — 같은 gradient, 다른 절댓값

Soft label은 teacher의 softmax 출력 — 모든 클래스에 0이 아닌 확률이 있는 분포입니다.

4.1 H(p)가 더 이상 0이 아님

예: teacher 출력 $p = (0.85, 0.10, 0.04, 0.01)$이면

$$ H(p) = -[0.85 \log 0.85 + 0.10 \log 0.10 + 0.04 \log 0.04 + 0.01 \log 0.01] \approx 0.53 \; \text{nats} $$

이 값은 학습 동안 일정 (teacher가 고정이므로).

4.2 Gradient는 여전히 같다

$$ \nabla_\theta H(p, q_\theta) = \nabla_\theta D_{\text{KL}}(p \| q_\theta) $$

따라서 학습 결과는 같음. 그럼 왜 굳이 KL을 쓰는가?

4.3 KL을 쓰는 실용적 이유 4가지

이유설명
(1) 의미 명확성 "이 loss는 분포 매칭이다"라는 의도가 코드에서 즉시 드러남. 미래의 코드 읽는 사람에게 신호
(2) Loss 값의 해석 KL은 항상 $\geq 0$이고 $p = q$일 때 정확히 0. 학습 진행을 시각적으로 추적하기 좋음. CE는 $H(p)$만큼 항상 양의 offset이 있어 "0에 얼마나 가까운가" 해석이 모호
(3) Library 인터페이스 PyTorch의 F.cross_entropy는 정답 인덱스를 받음 (분포 입력 불가). soft target을 쓰려면 F.kl_div를 호출해야 함
(4) 다중 distillation 손실 결합 Hinton KD의 $\mathcal{L} = (1-\alpha) \mathcal{L}_{\text{CE}} + \alpha T^2 \mathcal{L}_{\text{KL}}$ 처럼 두 항을 결합할 때, 두 항이 각자의 의미를 가지면 가중치 $\alpha$ 해석이 명확

5. PyTorch에서 실제로는?

PyTorch에는 두 손실이 분리되어 구현되어 있습니다. 입력 형식이 다르기 때문:

import torch
import torch.nn.functional as F

# === Hard label ===
# 정답: 클래스 인덱스 (정수 텐서)
logits = student(x)                  # (B, K)
hard_target = torch.tensor([3])      # (B,) — 인덱스
loss_hard = F.cross_entropy(logits, hard_target)
# 내부적으로 -log(softmax(logits)[hard_target]) 한 줄로 계산

# === Soft label ===
# 정답: teacher의 softmax 출력 (확률 분포)
teacher_logits = teacher(x).detach()                    # gradient 차단
log_q = F.log_softmax(logits / T, dim=-1)               # student log-prob
p     = F.softmax(teacher_logits / T, dim=-1)           # teacher prob
loss_soft = F.kl_div(log_q, p, reduction="batchmean")
# F.kl_div 시그니처: kl_div(log_q, p) — 첫 인자가 log-probability라는 점 주의
주의: F.kl_div의 인자 순서 PyTorch F.kl_div(input, target)는 input이 log-probability, target이 probability. 수학적 표기 $D_{\text{KL}}(p \| q)$와 비교하면 헷갈리기 쉬움:
  • 수학: $D_{\text{KL}}(\text{target} \| \text{prediction}) = \sum p_{\text{target}} \log(p_{\text{target}} / q_{\text{pred}})$
  • PyTorch: F.kl_div(log_q_pred, p_target) ← input은 log of student 분포, target은 teacher 분포
실수 흔함. 부호가 반대로 나오는 학습 결과의 90%는 이것 때문.

6. 그 유명한 $T^2$ 배수 — 왜 있는가?

Hinton KD의 최종 loss를 다시 보면:

$$ \mathcal{L} = (1 - \alpha) \cdot \mathcal{L}_{\text{CE}}(y, p_s) + \alpha \cdot \boxed{T^2} \cdot \mathcal{L}_{\text{KL}}(q_t, q_s) $$

$T^2$가 붙은 이유는 두 항의 gradient 크기를 비슷하게 맞추기 위해서입니다.

6.1 Temperature softmax의 logits

$q_s = \text{softmax}(z_s / T)$일 때, $z_s$에 대한 gradient는:

$$ \frac{\partial q_s}{\partial z_s} \propto \frac{1}{T} $$

이게 KL loss를 통해 한 번 더 흐르면 결국 student 파라미터에 대한 gradient는 $\frac{1}{T^2}$ 스케일.

6.2 보정

Hard label CE의 gradient는 $T$에 무관. 두 항이 균형을 이루려면 KL 항에 $T^2$를 곱해 gradient 스케일을 맞춤.

$T = 1$이면 $T^2 = 1$이라 영향 없음. $T = 5$이면 KL gradient가 25배 작아질 텐데, $T^2 = 25$를 곱해 보정.

통찰 $T^2$는 수학적 필연이 아니라 두 손실의 학습 신호 크기를 비슷하게 맞추는 실용 트릭. 일부 후속 연구는 다른 가중치(e.g., $T$, $\sqrt{T}$)를 쓰기도 함.

7. 그럼에도 두 손실이 "정말로" 다른 경우는?

7.1 KL의 비대칭성

$D_{\text{KL}}(p \| q) \neq D_{\text{KL}}(q \| p)$. 즉 어느 분포를 "정답" 자리에 놓느냐에 따라 다른 함수가 됨.

방향특성
$D_{\text{KL}}(p \| q)$
"Forward KL"		 $p$가 큰 곳에서 $q$가 작으면 강한 penalty. $q$가 $p$의 모든 mode를 cover하도록 학습 (mode-covering, mean-seeking)
$D_{\text{KL}}(q \| p)$
"Reverse KL"		 $q$가 큰 곳에서 $p$가 작으면 강한 penalty. $q$가 $p$의 일부 mode에만 집중 (mode-seeking)

KD에서는 보통 $D_{\text{KL}}(\text{teacher} \| \text{student})$ — student가 teacher 분포를 cover하도록.

7.2 수치적 안정성

  • $p_i = 0$인 항에서 $\log p_i$는 $-\infty$ → 직접 계산 시 NaN. PyTorch는 내부적으로 $0 \log 0 := 0$ 규약으로 처리
  • One-hot target에 KL을 직접 적용하면 위 문제 발생 가능. CE는 정답 인덱스 1개만 사용하므로 회피

7.3 Label smoothing의 관점

Hard label에 $\epsilon$만큼 smoothing을 적용하면:

$$ p_{\text{smooth}} = (1 - \epsilon) \cdot p_{\text{one-hot}} + \frac{\epsilon}{K} \cdot \mathbf{1} $$

이렇게 만든 $p_{\text{smooth}}$는 더 이상 one-hot이 아니라 soft label. $H(p_{\text{smooth}}) > 0$. 이 시점부터는 CE와 KL의 절댓값이 다름 (gradient는 여전히 같음).

통찰 Born-Again Networks(2018)에서 "self-distillation = label smoothing"이라는 주장이 나온 이유. soft target이 정확히 label smoothing 효과를 줌.

8. 정리 — 4가지 시나리오의 비교

시나리오 Target $H(p)$ CE KL 실용 추천
일반 분류 one-hot $y$ 0 $-\log q_y$ — 빠르고 직접 CE와 동일하지만 $K$개 항 형식 CE (효율)
Label smoothing soft (smooth된 one-hot) $> 0$ but 작음 분포 입력 가능한 CE 변형 필요 자연스럽게 처리 KL (혹은 soft-CE)
Hinton KD teacher softmax $> 0$ 종종 큼 가능하나 의미 모호 분포 매칭 의도 명확 KL (관습)
AD KD (US, RD4AD, EfficientAD) teacher feature (분포 아님) 적용 불가 적용 불가 MSE / cosine
AD 맥락에서의 주의 PatchCore·EfficientAD·RD4AD 등 AD 모델들의 distillation은 classification 분포가 아닌 raw feature 벡터를 매칭합니다. 따라서 CE/KL이 아니라 MSE ($\| T(x) - S(x) \|^2$) 또는 cosine distance를 사용. 본 문서의 CE vs KL 논의는 Hinton KD나 일반 분류 KD에 직접 적용되고, AD 모델에는 간접적으로만 적용됩니다.

9. 수식으로 한 번 더 — 학습 dynamics가 같음을 확인

$q_s = \text{softmax}(z_s)$가 student 출력, $p$가 teacher 출력일 때 student logit $z_s$에 대한 gradient:

9.1 KL의 경우

$$ \mathcal{L}_{\text{KL}} = \sum_i p_i \log \frac{p_i}{q_{s,i}} $$

$$ \frac{\partial \mathcal{L}_{\text{KL}}}{\partial z_{s,j}} = q_{s,j} - p_j $$

9.2 CE의 경우

$$ \mathcal{L}_{\text{CE}} = -\sum_i p_i \log q_{s,i} $$

$$ \frac{\partial \mathcal{L}_{\text{CE}}}{\partial z_{s,j}} = q_{s,j} - p_j $$

결정적 증거 두 gradient가 정확히 같음: $q_{s,j} - p_j$. 따라서 학습 dynamics는 완전히 동일. 두 손실의 값은 $H(p)$만큼 차이가 있지만, 학습에는 영향 없음.

10. 본 연구와의 연결

본 연구의 P1 IF-PROTO와 P2 CS-BRIDGE 모두 classification KD가 아닌 feature-level 매칭이라 본 분석의 CE/KL 구분은 직접 적용되지 않습니다. 그러나 다음 두 영역에서 간접적 시사:

본 연구 측면CE/KL 분석의 시사
P2 CS-BRIDGE의 pseudo-label 학습 MuSc/AnomalyCLIP 출력은 score (스칼라). 만약 분포 형태로 사용한다면 (e.g., per-patch class probability), KL 기반 학습이 자연스러움
Calibration (분석 ①의 P1 IF-PROTO 정정) Platt/Isotonic calibration도 결국 "예측 확률 분포를 정답 분포에 맞춤" — KL 관점이 자연스러움 (단 ranking은 invariant)
Student-Teacher score fusion 여러 모델의 softmax 분포를 ensemble할 때 KL 평균 또는 Jensen-Shannon divergence가 자연스러운 결합 metric

11. 한 줄 요약 (다시)

최종 정리 - 수학적으로: $H(p, q) = D_{\text{KL}}(p \| q) + H(p)$ — 두 손실은 $H(p)$만큼 차이지만 student gradient는 동일
- Hard label ($H(p) = 0$): CE = KL exactly. 효율 때문에 CE 사용
- Soft label ($H(p) > 0$): gradient는 같지만 의미·해석·라이브러리 인터페이스 때문에 KL 사용
- $T^2$ 배수: temperature softmax의 $1/T^2$ gradient scale 보정 — 두 항의 학습 신호 균형
- AD 모델들: 분포가 아닌 feature 매칭이라 CE/KL 대신 MSE 사용 — 본 토론과 별개 영역
