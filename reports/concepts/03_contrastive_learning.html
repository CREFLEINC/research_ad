0. 한 줄 답 — Contrastive Learning이란?

큰 그림 "라벨 없이도 좋은 표현을 학습할 수 있을까?"의 답. 핵심 아이디어: "같은 것은 가깝게, 다른 것은 멀게". 이미지의 두 가지 증강본(augmented views)이 같은 의미를 가진다고 가정하고, 같은 이미지의 두 view는 feature space에서 가깝게, 다른 이미지는 멀게 학습. 2018-2021년 사이 비약적 발전.

본 문서의 구성

  1. 2018-2019 — Instance Discrimination (InstDisc) — 개념의 시작
  2. 2020 SimCLR — 강한 augmentation + projection head + large batch
  3. 2020 MoCo — momentum encoder + memory queue (배치 의존성 해결)
  4. 2020 BYOL — negative 샘플 없이 학습 (충격적 발견)
  5. 2021 SimSiam — momentum도 빼고 stop-gradient만
  6. 이론 — InfoNCE = Mutual Information 하한
  7. AD 응용 — ReContrast (2023)에서 어떻게 쓰이는가

1. 왜 Contrastive Learning이 필요했는가

1.1 Supervised learning의 한계

ImageNet 1M장에 사람이 1000개 클래스를 라벨링한 결과로 ResNet, ViT 등이 학습됐습니다. 그러나:

  • 라벨 비용 — ImageNet 라벨링에 약 19,000시간 인력 투입 (Russakovsky et al. 2015)
  • 라벨 한계 — 1000개 클래스로 모든 시각 의미를 다 표현 못 함
  • 도메인별 재학습 — 의료 영상, 산업 검사 등 새 도메인마다 라벨링 필요

1.2 Self-supervised의 약속

"라벨 없이 데이터 자체에서 좋은 표현을 학습할 수 있다면 위 문제가 해결". 이미 NLP에서 BERT (2018), GPT (2018)가 self-supervised로 거대한 성공.

비전에서도 self-supervised의 흐름:

  • Pretext tasks (2015-2018): 직소퍼즐, 회전 예측, 색깔 복원 등
  • Contrastive learning (2018-2021): 표현 자체의 일관성 학습 — 본 문서 주제
  • Masked image modeling (2021+): MAE 등 BERT 스타일
Contrastive의 통찰 "이미지의 의미는 augmentation에 invariant해야 한다" — 같은 고양이를 자르거나 색을 바꾸거나 뒤집어도 여전히 고양이. 이 invariance를 학습 신호로 사용하면 라벨 없이도 의미 있는 feature를 얻을 수 있다.

2. 핵심 직관 — Pull Together, Push Apart

   이미지 1 (고양이)          이미지 2 (강아지)
        │                          │
   ┌────┴────┐                ┌────┴────┐
   ▼         ▼                ▼         ▼
   augA      augB             augA      augB
   (crop)    (color jitter)   (crop)    (flip)
   │         │                │         │
   ▼         ▼                ▼         ▼

   │         │                │         │
   ▼         ▼                ▼         ▼
   z₁ₐ      z₁ᵦ              z₂ₐ      z₂ᵦ
   └─── pull ────┘            └─── pull ────┘
   같은 이미지의 두 view        같은 이미지의 두 view
   = positive pair             = positive pair

                push
              z₁ₐ ←─── push away ───→ z₂ₐ
              z₁ₐ ←─── push away ───→ z₂ᵦ
              ...
              다른 이미지의 view = negative

   결과: feature space에서
   - 같은 이미지의 두 view는 가까워짐
   - 다른 이미지의 view는 멀어짐
   → 의미 비슷한 이미지들은 자연스럽게 cluster 형성
3가지 요소
  1. Anchor — 기준 sample (한 이미지의 한 view)
  2. Positive — anchor와 같은 이미지의 다른 view
  3. Negative — 다른 이미지의 view (수가 많을수록 학습 신호 강함)
Loss는 anchor가 positive에는 가까워지고 negative들로부터는 멀어지도록 설계.

3. InfoNCE Loss — 수식

거의 모든 contrastive learning이 사용하는 표준 손실. InfoNCE (Noise Contrastive Estimation) 또는 NT-Xent (Normalized Temperature-scaled Cross Entropy)라고 불립니다.

3.1 정의

Anchor $z_i$, positive $z_j$, $N$개 negative $z_k$가 있을 때:

$$ \mathcal{L}_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)} $$

여기서:

  • $\text{sim}(u, v) = \frac{u^\top v}{\|u\| \|v\|}$ — cosine similarity
  • $\tau$ — temperature parameter (보통 0.07 ~ 0.5)
  • $\mathbb{1}_{[k \neq i]}$ — anchor 자기 자신은 제외

3.2 직관 — Softmax over similarities

분자는 anchor-positive 유사도, 분모는 모든 sample과의 유사도 합. 결국 softmax 분류기와 같은 형태:

$$ \mathcal{L}_{NCE} = -\log p(\text{positive} | \text{anchor}) $$

"모든 sample 중에서 positive를 고를 확률을 최대화" → Cross-Entropy의 변형.

개념 ② 연결 InfoNCE는 cross-entropy의 친척. Anchor가 $z_i$, "정답"이 $z_j$ (positive), "오답"이 $z_k$ (negatives)인 $(2N-1)$-클래스 분류 문제로 볼 수 있음. 따라서 개념 ② KL/CE 가이드의 모든 통찰이 적용됨.

3.3 Temperature $\tau$의 역할

$\tau$ 값효과학습 양상
$\tau \to 0$가장 가까운 sample만 강조Hard negative mining 효과 ↑, 학습 불안정
$\tau \to \infty$모든 sample 균등 취급학습 신호 약화
$\tau = 0.07$ (전형)적당한 hard negative 가중SimCLR, MoCo 표준

Hinton KD의 temperature와 비슷한 역할이지만, 여기서는 "얼마나 강하게 negative를 멀리 밀어낼 것인가"를 조절.

4. 2018 — Instance Discrimination (시작)

Wu et al. "Unsupervised Feature Learning via Non-Parametric Instance Discrimination", CVPR 2018, arXiv:1805.01978

발상의 전환: "이미지 한 장 한 장을 모두 다른 클래스로 본다". ImageNet 1.2M 이미지가 있다면 1.2M-class 분류 문제 → 라벨 없이도 동작.

   학습 데이터: ImageNet 이미지 N장

   "정답" = 자기 자신
   "오답" = 다른 모든 이미지 (N-1개)

   각 이미지에 ID 부여: x_1, x_2, ..., x_N

   학습: f(x_i)가 x_i의 ID를 맞춰야 함
   loss = -log(softmax over all instances)

   결과: 각 이미지의 feature가 unique한 자기 위치를 학습
        + 비슷한 이미지는 자연스럽게 가까워짐 (general)

InstDisc의 한계

  • N이 너무 큼 — 1.2M에 대한 softmax 계산 불가능 → memory bank 도입 (각 이미지 feature 저장)
  • 여전히 단일 view — augmentation 활용 미흡 (SimCLR이 보완)

5. 2020 — SimCLR (간단함의 승리)

Chen et al. "A Simple Framework for Contrastive Learning of Visual Representations", ICML 2020, arXiv:2002.05709

5.1 핵심 아이디어 — 4가지 디자인 결정

구성SimCLR의 선택
Augmentation 강한 조합 — random crop + random color distortion + Gaussian blur. 둘 중 어느 하나만으로는 약함
Encoder ResNet-50
Projection head 비선형 2-layer MLP $g(\cdot)$ — encoder 출력에서 추가 변환. contrastive loss는 $g$ 출력에 적용
Loss NT-Xent (= InfoNCE) with temperature 0.07~0.5

5.2 구조

   원본 이미지 x
        │
   ┌────┴────┐
   ▼         ▼
   t(x)      t'(x)        ← 두 가지 augmentation t, t' (random sampling)
   = x̃ᵢ     = x̃ⱼ
   │         │
   ▼         ▼
   f(.)      f(.)         ← shared encoder (ResNet-50)
   │         │
   ▼         ▼
   hᵢ        hⱼ           ← representation (downstream task용)
   │         │
   ▼         ▼
   g(.)      g(.)         ← projection head (MLP)
   │         │
   ▼         ▼
   zᵢ        zⱼ           ← contrastive loss용

   배치 내 다른 이미지들의 z = negatives

   학습 후: g(.)를 버리고 f(.)만 downstream에 사용

5.3 핵심 발견 — Projection head의 중요성

놀라운 관찰 Projection head $g$ 없이 encoder 출력 $h$에 직접 contrastive loss를 적용하면 성능이 크게 떨어짐. 왜?

$g$를 통과한 $z$에는 contrastive 학습에 필요한 정보(예: augmentation invariance)만 남고, $h$에는 더 일반적 정보가 유지됨. $h$가 downstream에 더 유용.

5.4 SimCLR의 한계

한계설명
거대 batch 필요 negative가 같은 배치에서 옴 → batch 4096 이상 필요. 일반 GPU로 학습 불가
학습 시간 ↑ 큰 batch → 학습 epoch 늘림 (1000 epoch 권장)
Hardware 비용 TPU 또는 32+ GPU cluster 필요

이 한계 → MoCo가 해결.

6. 2020 — MoCo (Memory Queue + Momentum Encoder)

He et al. "Momentum Contrast for Unsupervised Visual Representation Learning", CVPR 2020, arXiv:1911.05722

6.1 핵심 아이디어 — Batch 의존성 제거

SimCLR은 batch 안에서만 negative를 얻음 → batch가 작으면 negative도 작음. MoCo는 memory queue로 이를 분리:

   현재 batch:                과거 K개 sample들 (queue)
   x_i (anchor)               z_1, z_2, ..., z_K
   x_j (positive)             ← negatives
        │
        ▼
   query encoder f_q          key encoder f_k (momentum copy)
        │                          │
        ▼                          ▼
   q                          k_+ (이번 batch positive)
                              │
                              ▼ enqueue
                          [k_+, k_K, k_{K-1}, ..., k_1]
                              │
                              ▼ dequeue (가장 오래된 제거)

   다음 step: queue의 모든 K+1 sample을 negative로 사용

6.2 Momentum Encoder — 안정성의 핵심

두 encoder를 사용:

  • Query encoder $f_q$ — 학습됨 (gradient descent)
  • Key encoder $f_k$ — gradient X. $f_q$ 가중치의 지수이동평균(EMA)로 천천히 따라감

$$ \theta_k \leftarrow m \theta_k + (1-m) \theta_q, \quad m \approx 0.999 $$

왜 momentum? Queue에 들어간 negative들은 과거 시점의 encoder가 만든 것. 만약 $f_k$가 빠르게 변하면 queue 안의 feature와 현재 query feature가 inconsistent해짐 → 학습 노이즈.

Momentum encoder는 천천히 변해서 queue의 일관성을 유지. m=0.999면 1000 step에서 절반 변화.

6.3 MoCo의 효과

측면SimCLRMoCo
Negative 수2N-1 (batch 내부)K ≈ 65,536 (queue)
Batch 요구4096+ 권장256으로 충분
HardwareTPU/32+GPU8 GPU로 학습 가능
ImageNet linear eval~71% (ResNet-50)~71% (비슷)

같은 성능을 훨씬 적은 자원으로 달성 — MoCo의 핵심 가치.

6.4 MoCo v2, v3 진화

  • MoCo v2 (2020) — SimCLR의 augmentation + projection head를 MoCo에 통합 (BBYRC, "Bring Best Yet, Reduce Cost")
  • MoCo v3 (2021) — ViT backbone, queue 제거 (큰 batch 다시 도입), self-supervised ViT의 첫 성공

7. 2020 — BYOL (Negative 없이도 된다!)

Grill et al. "Bootstrap Your Own Latent", NeurIPS 2020, arXiv:2006.07733

7.1 충격적 발견

BYOL의 주장 "Negative sample이 전혀 필요 없다." Anchor와 positive만 있어도 의미 있는 표현을 학습할 수 있다.

이게 왜 충격적인가? 직관적으로 negative가 없으면 모든 feature가 같은 값으로 수렴(collapse)하는 것이 당연. BYOL은 이를 어떻게 피했는가?

7.2 구조 — Online + Target Network

   이미지 x
   ┌─────┐
   ▼     ▼
   v     v'         ← 두 augmentation
   │     │
   ▼     ▼
   ┌─────┴────────────────────────┐
   │                              │
   ▼ Online network (학습됨)       ▼ Target network (EMA, frozen)
   f_θ                            f_ξ
   │                              │
   ▼                              ▼
   g_θ (projection)               g_ξ (projection)
   │                              │
   ▼                              ▼
   q_θ (predictor)  ←── MSE ──── stop_gradient(.)
   ↓
   loss = ||q_θ(v) - sg(g_ξ(v'))||²
                              ↑
                          stop gradient!

7.3 3가지 핵심 트릭

트릭역할
Predictor $q_\theta$ online 쪽에만 있는 추가 MLP. 비대칭 구조 — collapse 회피의 핵심
Target network (EMA) online의 momentum copy. 천천히 변해서 target이 "흔들리지 않음"
Stop-gradient target 쪽으로는 gradient 흐르지 않음. online만 학습

7.4 왜 collapse하지 않는가?

여전히 완전히 이해되지 않은 영역이지만, 가설:

  • 비대칭성 — predictor가 online-only이라 두 네트워크가 다른 방향으로 학습. 단순 "동일 출력으로 수렴"이 어려움
  • BN의 implicit 효과 — Batch Normalization이 mode collapse를 막는 implicit regularization 역할 (Richemond et al. 2020에서 일부 분석)
  • 학습 dynamics — predictor가 target distribution을 "예측"하는 게 핵심. random target에 fit하려는 학습이 collapse 회피

7.5 결과

방법Linear eval (ImageNet)Negative 필요?
SimCLR~69%O (batch)
MoCo v2~71%O (queue)
BYOL~74%X

Negative 없이도 더 좋은 성능. self-supervised learning의 새 시대.

8. 2021 — SimSiam (Momentum도 필요 없다)

Chen & He. "Exploring Simple Siamese Representation Learning", CVPR 2021, arXiv:2011.10566

8.1 더 단순화

BYOL의 trick 3개 중 무엇이 진짜 핵심인가? Chen & He의 ablation:

구성BYOLSimSiam
PredictorOO
Target network (EMA)OX (그냥 online의 copy)
Stop-gradientOO
NegativeXX

충격적 결론: Stop-gradient + predictor만으로도 학습 가능. Momentum encoder조차 필수가 아니다.

8.2 의미

SimSiam의 통찰 Collapse 회피의 진짜 열쇠는 비대칭성 (predictor) + stop-gradient. 다른 트릭들은 성능 향상용 옵션. 가장 단순한 형태로 contrastive learning이 가능.

그러나 SimSiam은 BYOL보다 약간 낮은 성능. 실용에서는 BYOL/MoCo가 여전히 표준.

9. InfoNCE의 이론적 의미 — Mutual Information 하한

InfoNCE는 단순한 분류 손실이 아닙니다. 정보이론적으로 깊은 의미를 가집니다.

9.1 명제

$N$개 negative와 1개 positive에 대한 InfoNCE는 anchor $X$와 positive $Y$의 mutual information $I(X; Y)$의 variational lower bound:

$$ I(X; Y) \geq \log N - \mathcal{L}_{NCE} $$

InfoNCE 최소화 = $I(X; Y)$ 최대화 (negative 수 $N$이 클수록 bound가 tight).

9.2 의미

"anchor와 positive 사이의 mutual information을 최대화" → 두 view가 공통으로 가진 정보(= 의미 있는 feature)를 보존. augmentation으로 사라지는 정보(예: 정확한 색깔, 위치)는 제거되고 의미 정보(예: 어떤 객체인가)는 보존.

개념 ②와의 연결 개념 ② §16.4에서 보았듯이 $I(X;Y) = D_{KL}(p(x,y) \| p(x)p(y))$. 즉 contrastive learning은 본질적으로 "결합 분포가 독립 분포에서 얼마나 멀리 떨어졌는가"를 최대화하는 KL 최대화 문제. 모든 길은 KL로 통한다.

참고: van den Oord et al., "Representation Learning with Contrastive Predictive Coding", 2018, arXiv:1807.03748 — InfoNCE의 이론 도입

10. AD에서의 응용 — ReContrast (2023)

Guo et al. "ReContrast: Domain-Specific Anomaly Detection via Contrastive Reconstruction", NeurIPS 2023, arXiv:2306.02602

10.1 동기

RD4AD (2022, 개념 ① §6)는 Teacher가 ImageNet pretrained라 domain gap이 있을 때 약함. ReContrast는 Teacher encoder를 target domain에 적응시키되 정상-이상 구분력을 유지하는 방법.

10.2 구조

   입력 x (정상 이미지만 학습)
        │
        ▼
   ┌─────────────────────────────────────────────┐
   │  Encoder E (학습됨, target domain에 적응)     │
   └─────────────────────────────────────────────┘
        │
        ├──── feature E(x) (target-adapted)
        │         │
        │         ▼ contrastive loss
        │  Source encoder E_src(x) (ImageNet pretrained, frozen)
        │  → E(x)와 E_src(x)가 의미적으로 유사하도록 제약
        │     (그러나 target domain 특성은 E에 흡수)
        │
        ▼
   ┌─────────────────────────────────────────────┐
   │  Decoder D (학습됨, RD4AD 스타일)             │
   └─────────────────────────────────────────────┘
        │
        ▼
   feature 재구성 → E(x)와의 mismatch로 anomaly 판정

10.3 핵심 — 두 가지 loss의 결합

$$ \mathcal{L}_{\text{ReContrast}} = \underbrace{\mathcal{L}_{RD}(E, D)}_{\text{reconstruction}} + \lambda \cdot \underbrace{\mathcal{L}_{\text{contrast}}(E, E_{\text{src}})}_{\text{domain 적응 regularizer}} $$

항역할
$\mathcal{L}_{RD}$ RD4AD 원래 loss — 정상 feature를 decoder가 재구성
$\mathcal{L}_{\text{contrast}}$ $E(x)$가 ImageNet pretrained $E_{\text{src}}(x)$와 비슷하도록 contrastive 제약. "적응하되 너무 멀리 가지 마라"

10.4 왜 contrastive loss인가?

ReContrast의 통찰 단순 MSE로 $E(x) = E_{\text{src}}(x)$를 강제하면 적응이 안 됨. Contrastive loss는 "유사하지만 동일하지 않은" 적응을 허용 — target domain에 맞게 일부 변화하되 의미 구조는 보존.

같은 이미지에 대한 두 encoder의 출력이 다른 이미지의 출력보다는 가까워야 한다는 InfoNCE 형태로 학습.

10.5 결과

MVTec AD: RD4AD 98.5% → ReContrast 99.5% (+1.0%p). Domain gap이 클수록 효과 ↑.

분석 ①에서 backbone adaptation 옵션의 (나)에 해당.

SimCLR (2020)O (large batch)XX (projection)X강한 augmentation + projection head

BYOL (2020)XOOOnegative 제거, predictor + EMA
SimSiam (2021)XXOOmomentum도 제거, stop-grad만

핵심 통찰들

  1. Augmentation이 모든 것의 기반 — 강하고 다양한 augmentation 없이는 어떤 contrastive도 작동 안 함
  2. Projection head는 정보 필터 — encoder는 일반 정보 유지, projection은 contrastive 학습 정보만
  3. Negative는 필수가 아니다 — BYOL이 보여줌. 비대칭성이 진짜 핵심
  4. Collapse 회피의 본질은 비대칭성 — predictor의 단방향, stop-gradient의 단방향 정보 흐름
  5. InfoNCE = MI 최대화 — 이론적으로 contrastive learning은 정보이론적으로 정당화됨

14. 큰 정리 (한 줄)

Contrastive Learning 9년 "같은 것은 가깝게, 다른 것은 멀게"라는 단순한 직관이 라벨 없는 표현 학습의 표준이 됨. 2018 InstDisc로 시작, 2020 SimCLR/MoCo가 표준화, 2020 BYOL가 negative 없이도 됨을 증명, 2021 SimSiam이 최소화. InfoNCE loss는 mutual information 하한이라는 이론적 정당성도 확보. AD 분야에서 ReContrast (2023)가 domain adaptation에 적용 — 본 연구의 비교 baseline 후보.
