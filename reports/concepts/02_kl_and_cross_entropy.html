KL Divergence와 Cross Entropy — 완전 가이드

작성일: 2026-05-28 (통합본) 관련: 개념 ① KD 흐름 학습 시간 60~90분

0. 한 줄 요약

큰 그림 $$ H(p, q) = D_{KL}(p \| q) + H(p) $$ 이 항등식이 모든 비밀의 출발점. Cross-Entropy는 "총 비용", KL은 "비효율 거리", Entropy는 "본질 비용".

학습 dynamics는 둘 다 동일 (gradient 같음). 그러나 의미·해석·도구 측면에서 차이가 있어 상황에 따라 선택. Hard label은 CE가 효율, Soft label은 KL이 깔끔.

Part I. 정의와 핵심 항등식

1. 세 가지 양의 정의

이산 분포 $p = (p_1, p_2, \ldots, p_K)$와 $q = (q_1, q_2, \ldots, q_K)$에 대해 ($\sum p_i = \sum q_i = 1$):

이름기호정의
Entropy$H(p)$$-\sum_i p_i \log p_i$
Cross-Entropy$H(p, q)$$-\sum_i p_i \log q_i$
KL Divergence$D_{KL}(p \| q)$$\sum_i p_i \log \frac{p_i}{q_i}$

$\log$는 자연로그 (nats) 또는 $\log_2$ (bits). 항등식 자체는 어떤 밑이든 성립.

관습:

  • $0 \log 0 := 0$ — 자세한 정당화는 Part II에서
  • $p_i > 0$이고 $q_i = 0$인 항에서 $D_{KL} = \infty$ — 경계 사례 §18에서

2. 측정 대상의 차이 — "둘 다 정보량 측정 아닌가?"

큰 틀에서는 맞지만 정밀하게 보면 측정 대상이 다릅니다.

양측정 대상의미
$H(p)$ $p$ 자체의 무질서도 "내가 가진 데이터에 본질적으로 있는 불확실성"
$H(p, q)$ $p$ 데이터를 $q$ 코드로 인코딩한 평균 비용 "잘못된 가정의 코드로 데이터를 표현한 총 비용 (절대값)"
$D_{KL}(p \| q)$ $p$와 $q$ 사이의 비효율 거리 "내가 잘못 가정해서 추가로 지불한 비용 (상대값, 0이 최적)"

비유 — 음식 배달 비용

요소의미
음식 자체 가격 = $H(p)$변할 수 없는 본질적 비용
배달비 = $D_{KL}(p \| q)$잘못된 배달 회사 (q)를 선택해서 추가로 든 비용. 직거래라면 0
총 지불 금액 = $H(p, q)$음식값 + 배달비 — 총 비용

학습 = 더 좋은 배달 회사 찾기. 즉 배달비($D_{KL}$)를 0에 가깝게. 학습으로 영향 줄 수 있는 부분은 배달비뿐이고, 음식값 ($H(p)$)은 외생적 고정.

정리 - CE는 혼합된 양: 본질 + 비효율
- KL은 순수한 양: 비효율만
- 같은 정보를 잰다기보다 같은 큰 그림 안의 서로 다른 부분을 잰다

3. 핵심 항등식 — 유도

$D_{KL}$ 정의에서 시작:

$$ D_{KL}(p \| q) = \sum_i p_i \log \frac{p_i}{q_i} $$

로그 분해 $\log(a/b) = \log a - \log b$:

$$ = \sum_i p_i \log p_i - \sum_i p_i \log q_i $$

각 항에 부호 정리:

$$ = -\underbrace{\Big( -\sum_i p_i \log p_i \Big)}_{H(p)} + \underbrace{\Big( -\sum_i p_i \log q_i \Big)}_{H(p, q)} $$

$$ D_{KL}(p \| q) = H(p, q) - H(p) $$

양변에 $H(p)$:

유도 완료 $$ \boxed{\; H(p, q) = D_{KL}(p \| q) + H(p) \;} $$ 가정 없음. 단순 로그 변형 + 항 재배열.

4. 정보이론적 의미 — 비트로 해석

4.1 Entropy $H(p)$ — 최소 평균 비트 수

분포 $p$를 따르는 데이터를 최적으로 인코딩할 때, 한 심볼당 평균 몇 비트가 필요한가?

Shannon의 source coding theorem: 심볼 $i$ (확률 $p_i$)에 $\log_2(1/p_i) = -\log_2 p_i$ 비트를 할당하는 것이 최적. 평균 비트:

$$ \mathbb{E}_{i \sim p} [-\log_2 p_i] = -\sum_i p_i \log_2 p_i = H_2(p) $$

$H(p)$의 정체 "$p$를 가장 잘 알고 가장 잘 코딩한다면 들어가는 평균 비트 수" — 데이터 자체의 본질적 정보량. 이보다 줄일 수는 없음.

4.2 Cross-Entropy $H(p, q)$ — 잘못된 코드의 평균 비트

데이터는 $p$를 따르는데 $q$가 맞다고 잘못 생각해서 $q$에 최적화된 코드를 쓴다면:

$$ \mathbb{E}_{i \sim p} [-\log_2 q_i] = -\sum_i p_i \log_2 q_i = H_2(p, q) $$

4.3 KL Divergence $D_{KL}$ — 추가 비트

$q$-코드는 $p$-코드보다 비쌈. 얼마나 더 비싼가?

$$ D_{KL}(p \| q) = \underbrace{H(p, q)}_{\text{잘못된 코드로 쓴 비트}} - \underbrace{H(p)}_{\text{최적 코드의 비트}} $$

4.4 항등식의 의미 — 비용 분해

$$ \underbrace{H(p, q)}_{\text{총 비용}} = \underbrace{D_{KL}(p \| q)}_{\text{비효율 페널티}} + \underbrace{H(p)}_{\text{최저 비용}} $$

   잘못된 q-코드로 인코딩한 총 비트 수
   ┌─────────────────────────────────────────────────┐
   │                                                 │
   │  ┌─ H(p) ─┐  ┌──── D_KL(p || q) ────┐          │
   │  │  최저  │  │   추가 비효율 비용     │          │
   │  │  비용  │  │   (이 코드 잘못 골라서) │          │
   │  └────────┘  └──────────────────────┘          │
   │                                                 │
   └─────────────────────────────────────────────────┘
        ↑                ↑
   "어쩔 수 없는"     "내가 잘못해서 추가로 든 비용"

핵심: $H(p)$는 데이터에 박힌 본질이라 학습으로 줄일 수 없는 부분. 학습이 줄일 수 있는 건 오직 $D_{KL}$.

5. Gibbs 부등식 — $D_{KL} \geq 0$ 증명

5.1 방법 1 — Jensen 부등식

$-\log$는 볼록함수. Jensen 부등식:

$$ D_{KL}(p \| q) = \mathbb{E}_p \Big[ -\log \frac{q_i}{p_i} \Big] \geq -\log \mathbb{E}_p \Big[ \frac{q_i}{p_i} \Big] = -\log \sum_i q_i = -\log 1 = 0 $$

등호는 $q_i/p_i$가 모든 $i$에서 같을 때 → $p = q$.

5.2 방법 2 — $\log x \leq x - 1$

모든 $x > 0$에 대해 $\log x \leq x - 1$ (등호 $x=1$):

$$ \log \frac{q_i}{p_i} \leq \frac{q_i}{p_i} - 1 $$

양변에 $-p_i$ 곱하고 합:

$$ D_{KL}(p \| q) = -\sum_i p_i \log \frac{q_i}{p_i} \geq -\sum_i (q_i - p_i) = 0 $$

Gibbs 부등식 결과 $D_{KL}(p \| q) \geq 0$이고 등호 ⟺ $p = q$. 학습 목표는 $D_{KL} \to 0$.

Part II. Hard Label (One-Hot)

7. One-hot에서 $H(p) = 0$인가? — $0 \log 0$의 부정형 문제

One-hot $p = (1, 0, 0, 0)$에 대해 entropy를 직접 계산하면:

$$ H(p) = -1 \cdot \log 1 \;-\; 0 \cdot \log 0 \;-\; 0 \cdot \log 0 \;-\; 0 \cdot \log 0 $$

항값문제
$-1 \cdot \log 1$$-1 \cdot 0 = 0$없음
$-0 \cdot \log 0$$-0 \cdot (-\infty)$부정형 — $0 \times \infty$은 미결정
핵심 의문 $0 \cdot (-\infty)$는 수학적 부정형 (indeterminate form). $0/0$, $\infty - \infty$와 같이 "고유한 답이 없는 식". 그래서 "$H(p) = 0$이다"라고 직접 말할 수 없고, 추가 규약이 필요.

7.1 두 가지 규약

Convention A — 표준 ML / 정보이론

$\boxed{\; 0 \log 0 := 0 \;}$ — 정의로 도입.

근거는 극한: $\lim_{x \to 0^+} x \log x = 0$ (L'Hôpital로 증명 가능).

$f(x) = x \log x$를 $x = 0$에서 연속 확장(continuous extension)하여 $f(0) := 0$.

Convention A 입장 "$0 \log 0$은 정의되지 않는 게 아니라, 연속 확장에 의해 정확히 0이다." 이 규약 하에서 $H(p)$, $D_{KL}$, $H(p,q)$가 모두 잘 정의되고, one-hot에서 $H(p) = 0$ 정확히.

Convention B — 엄격한 함수 해석학

$x \log x$의 정의역은 $x > 0$. $x = 0$에서 함수 정의 자체가 안 됨. One-hot에 대한 $H(p)$는 잘 정의되지 않음.

Convention B 입장 "엄밀히 말하면 one-hot에서 $H(p)$는 정의 안 됨. 극한값으로만 0에 수렴." 이 입장으로는 ML 실무가 진행 안 됨 → 결국 Convention A로 돌아옴.

어느 쪽이 옳은가?

둘 다 수학적으로 정합. 분야 관습:

  • 정보이론, ML, 통계학: Convention A 보편 사용
  • 엄격한 해석학 강의: Convention B 명시 후 연속 확장 별도 단계

본 문서 나머지는 Convention A 기준 (ML 실무 관행).

9. $\epsilon$-perturbation — "근사" 직관의 정량화

One-hot을 정확한 점이 아니라 "한없이 one-hot에 가까운 분포의 극한"으로 보면:

$$ p_\epsilon = \Big( 1-\epsilon, \; \tfrac{\epsilon}{3}, \; \tfrac{\epsilon}{3}, \; \tfrac{\epsilon}{3} \Big) $$

$\epsilon > 0$인 한 모든 원소 양수 → convention 불필요. $\epsilon \to 0$일 때 one-hot으로 수렴.

10. Hard label에는 CE를 쓴다 — 효율 이유

One-hot $p$일 때 $H(p,q)$ 계산을 풀어 쓰면:

$$ H(p_{\text{one-hot}}, q) = -\sum_i p_i \log q_i = -1 \cdot \log q_y = -\log q_y $$

정답 클래스의 예측 확률에 $-\log$ 한 번만. CE는 그 효율을 명시적으로 활용:

# CE: 인덱스 하나만 받아서 한 번의 로그 — 매우 효율적
loss_ce = -torch.log(q[y])

# KL: 형식상 K개 합을 계산 (대부분 0이지만)
loss_kl = torch.sum(p_onehot * (torch.log(p_onehot + eps) - torch.log(q)))

그리고 PyTorch의 F.cross_entropy(logits, target)는 target이 정수 인덱스 — 분포가 아닌 인덱스 1개만 받음. 이 효율 + 인터페이스 명확성이 hard label에 CE를 선택하는 이유.

Part III. Soft Label에서 KL을 쓰는 이유

11. Gradient 동일성 — 학습 dynamics는 같다

$q_s = \text{softmax}(z_s)$가 student 출력일 때 student logit $z_s$에 대한 gradient:

11.1 KL의 경우

$$ \mathcal{L}_{KL} = \sum_i p_i \log \frac{p_i}{q_{s,i}} $$

$$ \frac{\partial \mathcal{L}_{KL}}{\partial z_{s,j}} = q_{s,j} - p_j $$

11.2 CE의 경우

$$ \mathcal{L}_{CE} = -\sum_i p_i \log q_{s,i} $$

$$ \frac{\partial \mathcal{L}_{CE}}{\partial z_{s,j}} = q_{s,j} - p_j $$

결정적 증거 두 gradient가 정확히 같음: $q_{s,j} - p_j$. 학습 dynamics 동일. 두 손실의 값은 $H(p)$만큼 차이지만 학습에는 영향 없음.

왜 이렇게 되는가? 항등식 $H(p, q) = D_{KL}(p \| q) + H(p)$에서 $H(p)$는 target만의 함수, 학습 동안 고정. 따라서

$$ \nabla_\theta H(p, q_\theta) = \nabla_\theta D_{KL}(p \| q_\theta) + \underbrace{\nabla_\theta H(p)}_{= 0} = \nabla_\theta D_{KL}(p \| q_\theta) $$

12. 그래도 KL을 쓰는 4가지 실용 이유

12.1 의미적 명확성

# 신호: "분포 매칭을 한다"
loss = F.kl_div(log_q_student, p_teacher, reduction='batchmean')

# 신호: "분류기 학습을 한다"
loss = F.cross_entropy(student_logits, y_hard_label)

같은 일을 하더라도 의도가 다르면 함수를 분리. 코드 가독성.

   └──────────────────────────────→ epoch

12.3 다중 loss 결합 시 가중치 해석

Hinton KD 표준 loss:

$$ \mathcal{L} = (1-\alpha) \mathcal{L}_{\text{hard}} + \alpha \cdot T^2 \mathcal{L}_{\text{soft}} $$

옵션$\alpha$의 의미
둘 다 CE두 절댓값을 가중 평균. $\mathcal{L}_{\text{hard}}$는 floor 0 이고 $\mathcal{L}_{\text{soft}}$는 floor $H(p_{\text{soft}}) > 0$. $\alpha = 0.5$여도 "절반씩 기여" 아님 — soft 항이 항상 큼
Hard CE + Soft KL (표준)두 항 모두 "오차에 비례", 0이 정답. $\alpha$가 직관적으로 "soft 신호를 얼마나 신뢰하는가"

구체적 수치

$p_{\text{hard}}$ one-hot, $p_{\text{soft}}$ teacher ($H = 1.14$), 학생이 완벽 학습 시:

항CE+CECE+KL (표준)
$\mathcal{L}_{\text{hard}}$00
$\mathcal{L}_{\text{soft}}$$H(p_{\text{soft}}) = 1.14$ (CE)0 (KL)
$\alpha = 0.5$ 시 총합0.57 — 학습 완료인데 0 아님!0 — 깔끔

12.4 PyTorch API 강제

함수target 형식
F.cross_entropy(logits, target)정수 인덱스 (역사적 기본)
F.kl_div(log_q, p)분포 — soft target 가능

Soft target을 다루려면 KL이 자연스러운 인터페이스. (PyTorch 1.10+ F.cross_entropy가 분포 입력도 지원하지만, KL과 동등하고 관행은 KL.)

13. PyTorch 실전 코드

import torch
import torch.nn.functional as F

# Soft target (teacher output)
p = torch.tensor([0.7, 0.2, 0.07, 0.03])

# Student logits → softmax
z = torch.tensor([1.0, 0.5, 0.0, -0.3], requires_grad=True)
q = F.softmax(z, dim=-1)
log_q = F.log_softmax(z, dim=-1)

# === 방법 1: KL (표준) ===
loss_kl = F.kl_div(log_q, p, reduction='sum')
# → tensor(0.0876, grad_fn=...)

# === 방법 2: Soft CE (직접 계산) ===
loss_ce = -(p * log_q).sum()
# → tensor(1.0173, grad_fn=...)

# === 검증: H(p) ===
H_p = -(p * torch.log(p)).sum()
# → tensor(0.9297)

# loss_kl == loss_ce - H_p?
print(loss_ce.item() - H_p.item())  # 0.0876 — KL과 일치 ✓

# gradient 비교
loss_kl.backward(retain_graph=True)
grad_kl = z.grad.clone()
z.grad.zero_()
loss_ce.backward()
grad_ce = z.grad.clone()

print(torch.allclose(grad_kl, grad_ce))  # True ✓
PyTorch F.kl_div의 인자 순서 F.kl_div(input, target)는 input이 log-probability, target이 probability. 수학 표기 $D_{KL}(p \| q)$와 헷갈리기 쉬움:
  • 수학: $D_{KL}(\text{target} \| \text{prediction}) = \sum p_{\text{target}} \log(p_{\text{target}} / q_{\text{pred}})$
  • PyTorch: F.kl_div(log_q_pred, p_target) ← input은 log of student 분포, target은 teacher 분포
부호가 반대로 나오는 학습 결과의 90%는 이것 때문.
One-hot에 직접 KL = NaN 함정 
p = torch.tensor([1.0, 0.0, 0.0, 0.0])
H_naive = -(p * torch.log(p)).sum()  # → NaN! (0 * -inf in IEEE 754)
이론적으로 convention $0 \log 0 = 0$이지만 floating-point에서는 NaN. 학습 loss가 NaN으로 나오는 흔한 원인. One-hot target에 KL을 직접 적용하지 말고, F.cross_entropy를 쓰거나 masking으로 안전 처리.

Part IV. KD에서의 응용

14. Hinton KD의 표준 loss

$$ \mathcal{L} = (1 - \alpha) \cdot \mathcal{L}_{\text{CE}}(y, p_s) + \alpha \cdot T^2 \cdot \mathcal{L}_{\text{KL}}(q_t, q_s) $$

항역할
$\mathcal{L}_{\text{CE}}(y, p_s)$ 정답 label $y$에 대한 cross-entropy — student의 직접 분류 학습
$\mathcal{L}_{\text{KL}}(q_t, q_s)$ teacher $q_t$와 student $q_s$의 KL — soft target 매칭 (dark knowledge 전수)
$\alpha$ 두 신호의 가중치 (보통 0.5~0.9). KL을 쓰면 가중치 해석이 명확
$T^2$ 배수 temperature softmax의 $1/T^2$ gradient scale 보정 — 다음 §15

15. $T^2$ 배수의 이유

Temperature softmax $q = \text{softmax}(z/T)$는 student logit $z$에 대한 gradient에 $1/T$를 곱합니다:

$$ \frac{\partial q_i}{\partial z_j} \propto \frac{1}{T} $$

이게 KL loss를 통해 한 번 더 흐르면 student 파라미터에 대한 gradient는 $1/T^2$ 스케일.

15.1 보정

Hard label CE의 gradient는 $T$에 무관 ($T=1$ 사용). 두 항이 균형을 이루려면 KL 항에 $T^2$를 곱해 gradient 스케일을 맞춤.

  • $T = 1$이면 $T^2 = 1$ → 영향 없음
  • $T = 5$이면 KL gradient가 25배 작아질 텐데, $T^2 = 25$를 곱해 보정
통찰 $T^2$는 수학적 필연이 아니라 두 손실의 학습 신호 크기를 비슷하게 맞추는 실용 트릭. 일부 후속 연구는 다른 가중치(e.g., $T$, $\sqrt{T}$)를 쓰기도 함.

Part V. 확장 — 다른 ML loss와의 연결

16. 다른 ML loss들과의 연결

16.1 Maximum Likelihood Estimation (MLE)

데이터 분포 $p_{\text{data}}$에서 샘플로 모델 분포 $q_\theta$를 fit:

$$ \max_\theta \mathbb{E}_{x \sim p_{\text{data}}} [\log q_\theta(x)] = \min_\theta H(p_{\text{data}}, q_\theta) $$

즉 MLE = cross-entropy 최소화 = KL 최소화. 같은 일을 세 가지 이름.

16.2 Variational Inference / VAE

VAE의 ELBO: $\log p(x) \geq \mathbb{E}_{q(z|x)}[\log p(x|z)] - D_{KL}(q(z|x) \| p(z))$. KL이 prior와 posterior의 차이로 명시적 등장.

16.3 GAN의 발산

Original GAN: Jensen-Shannon divergence $\text{JS}(p \| q) = \frac{1}{2} D_{KL}(p \| m) + \frac{1}{2} D_{KL}(q \| m)$, $m = (p+q)/2$. KL의 대칭 변형.

16.4 Mutual Information

$I(X; Y) = D_{KL}(p(x,y) \| p(x)p(y))$. 두 변수의 결합 분포와 독립 분포의 KL. SimCLR, DIM 등 자기지도 학습의 이론 기초.

통합된 그림 ML loss의 대부분은 "두 분포 사이의 KL을 줄이자"의 다양한 변형. 항등식 $H(p,q) = D_{KL} + H(p)$가 그 출발점.

17. KL의 비대칭성 — Forward vs Reverse KL

$D_{KL}(p \| q) \neq D_{KL}(q \| p)$. 어느 분포를 "정답" 자리에 놓느냐에 따라 다른 함수.

방향특성
$D_{KL}(p \| q)$
Forward KL		 $p$가 큰 곳에서 $q$가 작으면 강한 penalty. $q$가 $p$의 모든 mode를 cover하도록 학습 (mode-covering, mean-seeking)
$D_{KL}(q \| p)$
Reverse KL		 $q$가 큰 곳에서 $p$가 작으면 강한 penalty. $q$가 $p$의 일부 mode에 집중 (mode-seeking)

KD에서는 보통 $D_{KL}(\text{teacher} \| \text{student})$ — student가 teacher 분포를 cover하도록.

비대칭성 시각화

$p = (0.9, 0.1)$, $q = (0.1, 0.9)$이면 우연히 $D_{KL}(p \| q) = D_{KL}(q \| p)$이지만, 일반적으로는 다름. 예: $p = (0.5, 0.5)$, $q = (0.99, 0.01)$:

  • $D_{KL}(p \| q) = 0.5 \ln(0.5/0.99) + 0.5 \ln(0.5/0.01) \approx 1.61$
  • $D_{KL}(q \| p) = 0.99 \ln(0.99/0.5) + 0.01 \ln(0.01/0.5) \approx 0.64$

방향에 따라 크게 다름. 따라서 KL은 거리(metric)가 아닌 비대칭 측도.

18. 경계 사례

18.1 $q_i = 0$이고 $p_i > 0$ — KL 발산

$\log(p_i / 0) = +\infty$. 정의 그대로 적용하면 $D_{KL} = +\infty$.

의미: "$q$가 어떤 가능성을 0으로 단정했는데, 실제 $p$에서는 그 가능성이 있다" → infinite penalty.

실무 대응 딥러닝의 softmax 출력은 절대 0이 되지 않음 (logit이 $-\infty$가 아닌 한). 그러나 floating-point underflow로 $q_i \approx 0$이 되어 $\log q_i = -\infty$가 될 수 있음. PyTorch의 log_softmax는 별도 함수로 제공 (수치 안정성 보장).

18.2 $p_i = 0$ — 관습 적용

$0 \cdot \log 0 := 0$. $0 \cdot \log q_i := 0$.

의미: "$p$가 0인 클래스는 합산에서 무시" — 그 클래스에 대해 $q$가 무엇이든 cross-entropy / KL 모두 영향 안 받음.

18.3 $p = q$ — 모든 양 최소

$$ D_{KL}(p \| p) = \sum_i p_i \log 1 = 0 $$

$$ H(p, p) = -\sum_i p_i \log p_i = H(p) $$

완전히 정확한 예측의 cross-entropy는 $H(p)$ — 0이 아니다 (one-hot 제외).

20. 큰 정리 (한 줄)

최종 $$ \underbrace{H(p, q)}_{\text{coding 비용}} = \underbrace{H(p)}_{\text{본질, 학습 불가}} + \underbrace{D_{KL}(p \| q)}_{\text{비효율, 학습으로 0 가능}} $$ - 학습 dynamics: CE/KL gradient 동일. 결과 같음
- Hard label: CE (효율). One-hot에서 convention 적용 시 exact, 그렇지 않으면 "근사"
- Soft label: KL (의미·모니터링·다중 loss·API 4가지 실용 이유)
- AD 모델들: 분포가 아닌 feature 매칭이라 MSE 사용 — 본 토론과 별개 영역
