# Tech Validation — 산업 현장 적용성 검증 절차

`tech-validator` 에이전트가 사용. 학술 SOTA와 산업 현장 사이의 간극을 정량적으로 식별한다.

## 1. 검증 4축

각 기법을 다음 4축으로 평가:

### A. 데이터 측면
- 학습에 필요한 **정상 샘플 수** (논문에서 보고된 N개 → 현장 확보 가능성)
- **결함 라벨 필요량** (unsupervised인지, weak label인지, full pixel mask인지)
- **도메인 시프트 강건성** — 조명/카메라/제품 변경 시 성능 저하율

### B. 성능 측면
- **AUROC만 보지 않는다.** 산업에서 중요한 지표:
  - `recall @ FPR=0.1%` (오검출 허용치가 매우 낮은 현장 기준)
  - 미세결함 검출 IoU
  - per-defect-type 분포 (특정 결함만 잘 잡는지)
- 학술 보고치 vs. 독립 재현치 차이

### C. 운영 측면
- 추론 latency (ms), FPS
- VRAM/RAM 요구량
- 에지 디바이스 호환성
- 모델 업데이트 주기 (정상 분포 변동 대응)

### D. 분포 변동
- production line의 정상 자체가 변동 (배치 차이, 계절성)
- 이를 처리하는 메커니즘 유무

## 2. 한계점 분류

각 한계는 3단계로:

| 분류 | 정의 | 예시 |
|------|------|------|
| **Blocker** | 현재 기술로 해결 불가, 새 접근 필요 | 결함 라벨 0장으로 새 결함 유형 학습 불가 |
| **Mitigable** | 추가 작업으로 완화 가능 | 데이터 증강 + 재학습으로 도메인 시프트 일부 해소 |
| **Acceptable** | 현장 운용 시 무시 가능 | 추론 200ms (사이클타임 1초 대비 충분) |

## 3. 확률적 평가 작성법

"이 방법의 산업 적용 성공 확률 약 N% — 근거: (1) ..., (2) ..., (3) ..."

근거는 반드시 다음 중 하나 이상:
- 벤치마크에서의 측정값
- 공정 요구 스펙과의 격차 (수치)
- 유사 도메인 보고 사례 수

## 4. 우선 극복 대상 (Top-3)

전체 한계 중 다음 기준으로 Top-3 선정:
- 산업 적용에 핵심적이고 (impact)
- 현재 SOTA로도 해결되지 않으며 (gap)
- 극복 시 파급 효과가 큰 것 (leverage)

이 Top-3가 연구팀의 다음 입력이 된다.

## 5. 검증 방법론 명시

보고서 마지막에 "검증을 위해 수행한 절차"를 기록 — 재현 시도, 자료 출처, 시간 제약 등.

- 격차 평가는 `tech-surveyor`가 수집한 수치를 기준으로 함
